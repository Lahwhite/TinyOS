## 背景知识：Shadow Map 的原理

Shadow Map 是最经典的实时阴影算法，分两个渲染 Pass：

### Pass 1：从光源视角渲染深度图

把摄像机放在光源位置，渲染整个场景，**只记录深度值**，不关心颜色。

这张深度图（Shadow Map）记录了："从光源看过去，场景中每个方向最近的物体距离是多少。"

### Pass 2：从摄像机视角正常渲染，判断阴影

正常渲染每个像素时，额外做一步：

1. 把这个像素的世界坐标变换到**光源的裁剪空间**（用 Pass 1 的 MVP）
2. 查询 Shadow Map：光源方向这个位置存的深度是多少（`shadowDepth`）
3. 比较当前像素到光源的距离（`currentDepth`）和 `shadowDepth`：
   - 如果 `currentDepth > shadowDepth + bias`：说明这个点在阴影里（有更近的物体挡住了光）
   - 否则：正常光照

```
                 光源
                  ↓
    ┌─────────────────────────────┐
    │  Pass 1：渲染深度图          │
    │                             │
    │  A(深度0.3)  B(深度0.5)     │
    │     ←── shadowMap记录这些   │
    └─────────────────────────────┘

    Pass 2：渲染摄像机视角
    像素C：从摄像机看到C点
           把C变换到光源空间 → 深度0.7
           查shadowMap对应位置 → 0.3（A挡住了）
           0.7 > 0.3 → C在阴影中 → 颜色变暗
```

### Bias（深度偏移）

直接比较深度会产生**自遮挡（Shadow Acne）**——表面会出现条纹状的错误阴影。原因是浮点精度问题，一个表面可能会遮挡自己。

解决方法是加一个小偏移：
```
if (currentDepth > shadowDepth + bias) → 在阴影里
```
`bias` 通常取 `0.005` ~ `0.05`，值太大阴影会和物体脱离（Peter Pan Shadow），值太小条纹还在。

// 返回 zbuffer[y * width + x]，越界返回 1.0f（最远）
float getDepth(int x, int y) const;

// 获取 framebuffer 宽高（Shadow Map 采样时需要转换坐标）

**2. 在 `framebuffer.cpp` 中实现**

```cpp
float Framebuffer::getDepth(int x, int y) const {
    if (x < 0 || x >= width || y < 0 || y >= height) return 1.0f;
    return zbuffer[y * width + x];
}
int Framebuffer::getWidth()  const { return width; }
int Framebuffer::getHeight() const { return height; }
```

## 实验二：Pass 1 —— 从光源视角渲染深度图

### 目标
用光源视角的 MVP 矩阵渲染整个场景，把深度写入一个单独的 `Framebuffer`，得到 Shadow Map。

### 背景知识：光源的 MVP

把光源当作一个摄像机，构造它的 MVP 矩阵：

- **Model 矩阵**：和正常渲染相同（`Mat4::identity()`）
- **View 矩阵**：`Mat4::lookAt(lightPos, center, up)`，光源位置看向场景中心
- **Projection 矩阵**：可以用**正交投影**（平行光适合），也可以用透视投影（点光源适合）

african_head 场景用**正交投影**更合适（模拟平行的太阳光）。正交投影矩阵比透视矩阵简单：

```
orthographic(l, r, b, t, near, far)：

| 2/(r-l)     0       0    -(r+l)/(r-l) |
|    0     2/(t-b)    0    -(t+b)/(t-b) |
|    0        0    -2/(f-n) -(f+n)/(f-n)|
|    0        0       0         1       |
```

对于 african_head 这种归一化的模型（坐标在 [-1,1]），可以用：
`l=-1, r=1, b=-1, t=1, near=-1, far=1`（把整个模型包在正交体里）

> **注意：** 正交投影下透视除法的 w=1，`toVec3` 直接取 xyz 即可，不会有透视变形。

**在 `mat.h` 中新增正交投影矩阵：**

```cpp
static Mat4 orthographic(float l, float r, float b, float t, float zNear, float zFar) {
    Mat4 result;
    result.m[0][0] = 2.0f / (r - l);
    result.m[1][1] = 2.0f / (t - b);
    result.m[2][2] = -2.0f / (zFar - zNear);
    result.m[0][3] = -(r + l) / (r - l);
    result.m[1][3] = -(t + b) / (t - b);
    result.m[2][3] = -(zFar + zNear) / (zFar - zNear);
    result.m[3][3] = 1.0f;
    return result;
}
```

### 步骤

**在 `main.cpp` 中实现 Pass 1**

```
1. 创建 shadowMap（一个单独的 Framebuffer，尺寸和主 framebuffer 相同）
2. 构造光源 MVP：
   - lightPos 与 lightDir 对应（光从 lightPos 朝原点照）
   - 用正交投影
3. 遍历所有面，用 drawTriangle(Vec3, Vec3, Vec3, Color) 渲染
   （只需要写深度，颜色随便填，用第三阶段的纯色版本即可）
4. Pass 1 结束，shadowMap 的 zbuffer 就是 Shadow Map
```

示例结构：

```cpp
// --- Pass 1：生成 Shadow Map ---
Vec3 lightPos(1.0f, 1.0f, 1.0f);          // 光源位置
Vec3 lightDir = Vec3::normalize(lightPos); // 光照方向

Framebuffer shadowMap(512, 512);
shadowMap.clearZBuffer();

Mat4 lightModel = Mat4::identity();
Mat4 lightView  = Mat4::lookAt(lightPos, Vec3(0,0,0), Vec3(0,1,0));
Mat4 lightProj  = Mat4::orthographic(-1.0f, 1.0f, -1.0f, 1.0f, -1.0f, 1.0f);
Mat4 lightMVP   = lightProj * lightView * lightModel;

for (int i = 0; i < model.numFaces(); ++i) {
    Vec3 sp0 = vertTransform(model.vert(i, 0), lightMVP, 512, 512);
    Vec3 sp1 = vertTransform(model.vert(i, 1), lightMVP, 512, 512);
    Vec3 sp2 = vertTransform(model.vert(i, 2), lightMVP, 512, 512);
    shadowMap.drawTriangle(sp0, sp1, sp2, Color{255,255,255});  // 颜色无所谓
}
// 此时 shadowMap 的 zbuffer 就是 Shadow Map，可以用
```

## 实验三：Pass 2 —— 阴影判断

### 目标
在正常渲染的每个像素处，查询 Shadow Map，判断该点是否在阴影中，据此调整光照强度。

### 核心问题：如何查询 Shadow Map

Pass 2 渲染某个像素时，已知：
- 该像素对应的世界坐标（顶点坐标，经过重心坐标插值得到）
- 光源的 MVP 矩阵（`lightMVP`）

查询步骤：
```
1. 把世界坐标变换到光源裁剪空间：
   Vec4 lightClip = lightMVP * toVec4Point(worldPos)

2. 透视除法得到光源 NDC：
   Vec3 lightNDC = toVec3(lightClip)

3. 把 NDC 映射到 shadowMap 的像素坐标（同 vertTransform 的最后一步）：
   int sx = (int)((lightNDC.x + 1.0f) / 2.0f * shadowMap.getWidth())
   int sy = (int)((lightNDC.y + 1.0f) / 2.0f * shadowMap.getHeight())

4. 查询 shadowMap 中该位置的深度：
   float shadowDepth = shadowMap.getDepth(sx, sy)

5. 当前点到光源的深度（光源 NDC 的 z 值）：
   float currentDepth = lightNDC.z

6. 判断：
   bool inShadow = (currentDepth > shadowDepth + bias)
```

### 步骤

**1. 在 `framebuffer.h` 中新增带阴影的 drawTriangle 重载**

在现有最完整的光照版本基础上，新增两个参数：`lightMVP`（光源变换矩阵）和 `shadowMap`（引用）。

```cpp
void drawTriangle(
    const Vec3& p0, const Vec3& p1, const Vec3& p2,
    const Vec2& uv0, const Vec2& uv1, const Vec2& uv2,
    const Vec3& n0,  const Vec3& n1,  const Vec3& n2,
    const Texture& diffuseTex,
    const Vec3& lightDir,
    const Vec3& viewDir,
    const Texture& normalMap,
    const Mat4& lightMVP,           // 新增：光源 MVP
    const Framebuffer& shadowMap    // 新增：Shadow Map
);
```

> **注意：** `Framebuffer` 引用自身，需要前向声明或确保 include 顺序正确。由于 `drawTriangle` 的参数是 `const Framebuffer&`，而这个函数本身也是 `Framebuffer` 的成员，C++ 允许在类内使用自身类型的引用作为参数，不需要额外处理。

**2. 在 `framebuffer.cpp` 中实现**

从上一个 `drawTriangle` 复制代码，在写入颜色之前增加阴影判断：

```
在计算 u/v、采样法线、Blinn-Phong 之前，先插值世界坐标（需要把世界坐标也传进来
或者近似处理——见下面的说明）。

最简单的近似方案：不插值精确世界坐标，直接用屏幕坐标对应的 NDC 坐标反查。

更正确的方案（推荐）：在 drawTriangle 里也传入三个顶点的世界坐标（wv0/wv1/wv2），
用重心坐标插值得到当前像素的世界坐标，再做 lightMVP 变换。
```

**推荐方案：传入世界坐标**

把函数签名改为同时接受屏幕坐标（p0/p1/p2）和世界坐标（w0/w1/w2）：

```cpp
void drawTriangle(
    const Vec3& p0,  const Vec3& p1,  const Vec3& p2,   // 屏幕坐标+深度
    const Vec3& w0,  const Vec3& w1,  const Vec3& w2,   // 世界坐标（新增）
    const Vec2& uv0, const Vec2& uv1, const Vec2& uv2,
    const Vec3& n0,  const Vec3& n1,  const Vec3& n2,
    const Texture& diffuseTex,
    const Vec3& lightDir,
    const Vec3& viewDir,
    const Texture& normalMap,
    const Mat4& lightMVP,
    const Framebuffer& shadowFB
);
```

在像素循环里：
```cpp
// 插值世界坐标
Vec3 worldPos(
    alpha * w0.x + beta * w1.x + gamma * w2.x,
    alpha * w0.y + beta * w1.y + gamma * w2.y,
    alpha * w0.z + beta * w1.z + gamma * w2.z
);

// 变换到光源空间
Vec4 lightClip  = lightMVP * toVec4Point(worldPos);
Vec3 lightNDC   = toVec3(lightClip);
int sx = (int)((lightNDC.x + 1.0f) / 2.0f * shadowFB.getWidth());
int sy = (int)((lightNDC.y + 1.0f) / 2.0f * shadowFB.getHeight());
float shadowDepth   = shadowFB.getDepth(sx, sy);
float currentDepth  = lightNDC.z;

float shadowFactor = (currentDepth > shadowDepth + 0.02f) ? 0.3f : 1.0f;
// shadowFactor = 0.3 表示在阴影里（只保留30%亮度），1.0 表示正常光照

// 在 Blinn-Phong 结果上乘以 shadowFactor（只乘漫反射和高光，不乘环境光）：
float r = std::min(1.0f, texColor.r / 255.0f * (ambient + diff * shadowFactor)
                       + spec * 0.5f * shadowFactor);
// g/b 同理
```

**3. 修改 `main.cpp`，串联两个 Pass**

```cpp
// Pass 2 渲染循环
for (int i = 0; i < model.numFaces(); ++i) {
    Vec3 w0 = model.vert(i, 0);   // 世界坐标
    Vec3 w1 = model.vert(i, 1);
    Vec3 w2 = model.vert(i, 2);

    Vec3 p0 = vertTransform(w0, mvp, 512, 512);  // 屏幕坐标
    Vec3 p1 = vertTransform(w1, mvp, 512, 512);
    Vec3 p2 = vertTransform(w2, mvp, 512, 512);

    Vec2 uv0 = model.uv(i, 0);
    Vec2 uv1 = model.uv(i, 1);
    Vec2 uv2 = model.uv(i, 2);

    Vec3 n0 = model.normal(i, 0);
    Vec3 n1 = model.normal(i, 1);
    Vec3 n2 = model.normal(i, 2);

    framebuffer.drawTriangle(p0, p1, p2, w0, w1, w2,
                             uv0, uv1, uv2,
                             n0, n1, n2,
                             texture, lightDir, viewDir, normalMap,
                             lightMVP, shadowMap);
}
```

## 常见问题

**Q：阴影完全没有出现（全部不在阴影里）？**  
A：按以下顺序排查：
1. 临时把 `shadowFactor` 硬编码为 `0.3f`（不做阴影判断），确认着色流程本身正确
2. 输出 shadowMap 可视化（实验四第4步），确认 Pass 1 写入了深度
3. 检查 `lightMVP` 的构造：`lightProj * lightView * lightModel` 顺序是否正确
4. 打印几个像素的 `currentDepth` 和 `shadowDepth`，看数值范围是否合理

**Q：整个模型都在阴影里（全部变暗）？**  
A：`bias` 太小或者深度比较方向反了。检查判断条件：`currentDepth > shadowDepth + bias` 是正确的（当前深度大于 shadowMap 存的深度，说明有物体在前面挡住了光）。如果反过来写成 `<`，所有点都会误判为阴影。

**Q：阴影有条纹（Shadow Acne）？**  
A：增大 `bias`，从 `0.005f` 逐步增大到 `0.05f`，找到条纹消失的最小值。

**Q：阴影边缘有明显锯齿？**  
A：这是 Shadow Map 分辨率有限的固有问题。解决方案是增大 shadowMap 的分辨率（比如 1024×1024），或者使用 PCF（Percentage Closer Filtering）——对 shadowMap 中周围多个像素采样取平均，得到软边缘阴影。这是进阶优化，不是必须的。

**Q：正交投影矩阵的参数怎么设置？**  
A：l/r/b/t 决定了正交视锥体的范围，要把场景中所有物体都包含在内。african_head 坐标在 [-1,1]，所以 `l=-1, r=1, b=-1, t=1` 刚好合适。near/far 也要包住所有 z 值，用 `-1` 到 `1` 或者 `-2` 到 `2` 都可以。
