## 背景知识：为什么需要 Z-buffer？

第二阶段画线框时，所有边都被画出来了，包括模型背面的边——你可以看到线穿透了头部。这是因为没有任何遮挡处理。

改成实心三角形之后，问题变得更严重：如果先画远处的三角形，再画近处的三角形，结果是对的；但如果顺序反过来，近处的三角形会被远处的覆盖，产生错误的画面。

解决办法就是 **Z-buffer（深度缓冲）**：

- 维护一个和像素缓冲大小相同的浮点数组，每个位置存储"当前写入该像素的最近深度值"
- 初始化为最远值（`+∞` 或 `1.0f`，取决于深度约定）
- 每次准备写一个像素时，先检查：**当前片元的深度 是否比缓冲里存的深度更近？**
  - 是：更新像素颜色，同时更新深度缓冲
  - 否：丢弃，不写入

这样无论三角形以什么顺序提交，最终每个像素都只保留最近的那个。

---

## 深度值的范围约定

经过透视投影和透视除法之后，NDC 的 z 值范围是 `[-1, 1]`：
- `-1` 对应近裁剪面（最近）
- `+1` 对应远裁剪面（最远）

因此深度缓冲初始化为 `1.0f`（最远），每次判断时：**z 值越小，越近**。

## 实验一：重构 Framebuffer，加入深度缓冲

### 目标
把像素存储从二维 `vector<vector<Color>>` 改成一维 `vector<Color>`，同时新增一个 `vector<float>` 作为深度缓冲。

### 为什么要改成一维？

二维 `vector<vector<Color>>` 的每一行在内存中是分散的，CPU 读取时缓存命中率低。一维数组在内存中完全连续，是渲染器的标准做法。

### 步骤

**1. 修改 `framebuffer.h`**

私有成员从：
```cpp
std::vector<std::vector<Color>> pixels;
```
改为：
```cpp
std::vector<Color> pixels;    // 一维像素缓冲，下标 = y * width + x
std::vector<float> zbuffer;   // 深度缓冲，与 pixels 等大
```

同时新增深度缓冲相关的公有接口：
```cpp
void clearDepth();   // 把深度缓冲重置为 1.0f（每帧开始前调用）
```

另外，为第三阶段新增一个支持深度测试的三角形绘制函数（旧的 `drawTriangle` 保留不动，新增一个重载）：
```cpp
// 新增：带深度插值和深度测试的三角形，v0/v1/v2 是屏幕坐标+深度
void drawTriangle(const Vec3& v0, const Vec3& v1, const Vec3& v2, const Color& color);
```

> **为什么新增重载而不修改原来的？**  
> 旧的 `drawTriangle(Vec2, Vec2, Vec2, Color)` 接受二维坐标，第一阶段的测试代码还在用它。保留旧接口不破坏已有代码，新接口用三维坐标（x, y 是屏幕坐标，z 是深度值）。C++ 函数重载会根据参数类型自动区分，不会冲突。

**2. 修改 `framebuffer.cpp`**

**构造函数：** 像素存储从二维改为一维：
```
pixels 大小改为 width * height
zbuffer 大小同样为 width * height，初始化为 1.0f
```

**`setPixel`：** 下标计算从 `pixels[y][x]` 改为 `pixels[y * width + x]`

**`writeTGA`：** 遍历顺序不变，下标计算同上改为一维访问

**`clearDepth`：** 把 `zbuffer` 所有元素重置为 `1.0f`：
```cpp
std::fill(zbuffer.begin(), zbuffer.end(), 1.0f);
```

**3. 实现新的 `drawTriangle(Vec3, Vec3, Vec3, Color)`**

这是本阶段的核心。函数逻辑与第一阶段的 `drawTriangle` 几乎一样，只是多了深度插值和深度测试两步。

算法流程：

```
1. 计算包围盒（与第一阶段相同，用 v0.x/v1.x/v2.x 的 x 分量计算）

2. 遍历包围盒内的每个像素 (x, y)

3. 计算重心坐标 (α, β, γ)（与第一阶段完全相同）

4. 如果 α < 0 || β < 0 || γ < 0，跳过

5. 【新增】插值深度值：
   float z = α * v0.z + β * v1.z + γ * v2.z

6. 【新增】深度测试：
   int idx = y * width + x
   if (z >= zbuffer[idx]) continue;   // 当前片元更远，丢弃
   zbuffer[idx] = z;                  // 更近，更新深度缓冲

7. setPixel(x, y, color)              // 写入颜色
```

> **重心坐标插值的含义：**  
> α、β、γ 是三个顶点的权重，三个顶点的深度值 `v0.z`、`v1.z`、`v2.z` 按这个权重加权平均，就得到了当前像素的深度值。这是光栅化中"属性插值"的核心思想——后续阶段的颜色、UV 坐标、法线都用完全相同的方式插值。

### 检查点
- [ ] 编译通过，原有测试（第一阶段的二维三角形）仍然正常
- [ ] `clearDepth()` 能正常调用
- [ ] 新的 `drawTriangle(Vec3, Vec3, Vec3, Color)` 能正常调用

## 实验二：升级顶点变换，返回深度值

### 目标
修改 `main.cpp` 中的 `vertTransform` 函数，让它同时返回屏幕坐标和深度值（即返回 `Vec3` 而不是 `Vec2`）。

### 步骤

把 `vertTransform` 的返回类型从 `Vec2` 改为 `Vec3`：

```cpp
Vec3 vertTransform(const Vec3& worldPos, const Mat4& mvp, int screenWidth, int screenHeight) {
    Vec4 clip = mvp * toVec4Point(worldPos);
    Vec3 ndc  = toVec3(clip);                         // 透视除法，含 x/y/z
    float screenX = (ndc.x + 1.0f) / 2.0f * screenWidth;
    float screenY = (ndc.y + 1.0f) / 2.0f * screenHeight;
    float screenZ = ndc.z;                             // 保留 NDC 的 z，范围 [-1, 1]
    return Vec3(screenX, screenY, screenZ);
}
```

返回的 `Vec3` 中：
- `x`、`y`：像素坐标（传给 `drawTriangle` 的屏幕坐标）
- `z`：深度值（传给 `drawTriangle` 用于深度测试）

### 检查点
- [ ] `vertTransform` 返回类型改为 `Vec3`，编译通过

**2. 改写主渲染循环**

新的渲染主流程：

```
1. 创建 Framebuffer（黑色背景，不需要手动刷背景色）
```

## 实验四：加入简单的面法线着色（可选，强烈建议做）

### 目标
根据三角面的朝向（面法线）计算一个简单的光照颜色，让模型有明暗变化，不再是纯白色。

这不是完整的光照模型（第五阶段会做），但只需要几行代码，效果提升非常明显，而且能帮助你提前理解法线和光照的关系。

### 原理

面法线 = 三角形平面的垂直方向向量：

```
边1 = v1 - v0
边2 = v2 - v0
面法线 n = 边1 × 边2（叉积），然后归一化
```

光照强度 = 面法线与光照方向的点积：

```
light_dir = Vec3(0, 0, -1)   // 光从摄像机方向射来（z 轴负方向）
intensity = n · light_dir     // 点积，范围 [-1, 1]
```

点积的含义：法线越朝向光源，点积越大（越亮）；法线背离光源，点积为负（背面）。

颜色映射：

```
if (intensity <= 0)  跳过这个面（背面，不渲染）
color = Color{ uint8_t(255 * intensity), uint8_t(255 * intensity), uint8_t(255 * intensity) }
```

这样正面会有 0~255 的灰度变化，背面直接跳过不渲染。

### 步骤

在主渲染循环里，每个面渲染前计算一下面法线和光照强度：

```cpp
// 用世界坐标的顶点计算面法线（不用变换后的屏幕坐标）
Vec3 wv0 = model.vert(i, 0);
Vec3 wv1 = model.vert(i, 1);
Vec3 wv2 = model.vert(i, 2);
Vec3 n = Vec3::normalize((wv1 - wv0).cross(wv2 - wv0));

Vec3 lightDir(0.0f, 0.0f, -1.0f);
float intensity = n.dot(lightDir);

if (intensity <= 0) continue;   // 背面剔除

Color color{ uint8_t(255 * intensity),
             uint8_t(255 * intensity),
             uint8_t(255 * intensity) };

// 再做变换，画三角形
Vec3 sv0 = vertTransform(wv0, mvp, 512, 512);
...
framebuffer.drawTriangle(sv0, sv1, sv2, color);
```

> **注意：** 面法线用**世界坐标**的顶点计算，不是变换后的屏幕坐标。屏幕坐标已经经过投影扭曲，用它计算法线会得到错误结果。

## 常见问题

**Q：画出来的人头是空的（只有轮廓，没有填充）？**  
A：检查新的 `drawTriangle(Vec3, Vec3, Vec3, Color)` 是否正确实现，确认调用的是这个版本而不是旧的二维版本。

**Q：遮挡关系还是错的（背面穿透）？**  
A：检查深度测试的判断方向：应该是 `z >= zbuffer[idx]` 时丢弃（当前片元更远）。如果写成了 `<=`，逻辑是反的。另外确认 `clearDepth()` 在渲染循环前被调用了。

**Q：整个画面全黑？**  
A：检查深度缓冲初始化是否是 `1.0f`。如果初始化为 `0.0f`，所有深度测试都会失败（NDC 的 z 在 [-1,1] 范围，而 0.0f 比大多数片元的 z 小），导致所有像素都被丢弃。

**Q：做了面法线着色，但模型全黑？**  
A：检查 `intensity <= 0` 的判断，以及 `uint8_t(255 * intensity)` 的转换。如果 `intensity` 很小（比如 0.01），颜色值会是 `uint8_t(2)`，显示出来接近黑色但不是全黑，可以先打印几个 intensity 值确认范围。另外确认光照方向 `lightDir = (0, 0, -1)` 和你的模型朝向一致（african_head 默认面朝 +z 方向）。

**Q：包围盒裁剪了吗？**  
A：新的 `drawTriangle(Vec3, Vec3, Vec3, Color)` 里记得把包围盒限制在屏幕范围内（`xmin = max(xmin, 0)`，`xmax = min(xmax, width-1)`，y 同理），否则 `zbuffer` 的下标可能越界。
