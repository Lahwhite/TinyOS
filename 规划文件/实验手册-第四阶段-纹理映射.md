## 背景知识：纹理映射是什么

纹理映射做的事情是：把一张 2D 图片"贴"到 3D 模型的表面。

每个三角形的顶点除了有 3D 坐标，还有一个 **UV 坐标**（也叫纹理坐标），表示这个顶点对应纹理图片上的哪个位置：

- `u`：水平方向，范围 `[0, 1]`（0 是最左，1 是最右）
- `v`：垂直方向，范围 `[0, 1]`（0 是最下，1 是最上）

光栅化时，三角形内部每个像素的 UV 坐标通过**重心坐标插值**得到（和第三阶段插值深度值的方法完全一样），然后用插值出的 UV 去纹理图片上采样颜色。

```
顶点 v0 的 UV = (0.1, 0.2)
顶点 v1 的 UV = (0.8, 0.1)
顶点 v2 的 UV = (0.5, 0.9)

像素 P 的重心坐标 = (α=0.3, β=0.5, γ=0.2)
像素 P 的 UV = 0.3*(0.1,0.2) + 0.5*(0.8,0.1) + 0.2*(0.5,0.9)
             = (0.03+0.40+0.10, 0.06+0.05+0.18)
             = (0.53, 0.29)

去纹理图片的 (0.53, 0.29) 位置取颜色 → 像素 P 的颜色
```

## 实验一：准备纹理文件，实现 Texture 类

### 目标
下载纹理文件，新建 `Texture` 类，能从 TGA 文件中读取像素，并根据 UV 坐标返回颜色。

### 步骤

**1. 下载纹理文件**

从 tinyrenderer 的仓库下载 `african_head_diffuse.tga`，放到 `game/assets/` 目录下：

```
https://github.com/ssloy/tinyrenderer/tree/master/obj/african_head
```

下载 `african_head_diffuse.tga`（约 1024×1024 的彩色图片）。

**2. 新建 `src/core/types.h`，把 Color 移入**

`Color` 目前定义在 `framebuffer.h` 里，但 `texture.h` 需要用它，而 `framebuffer.h` 后续又要 include `texture.h`，会形成循环依赖。

正确做法是把 `Color` 提取到一个独立的基础类型文件，职责清晰，任何人都可以 include 它而不引入循环：

```cpp
// src/core/types.h
#pragma once

#include <cstdint>

struct Color { uint8_t r, g, b; };
```

然后做以下改动：
- **`framebuffer.h`**：把 `struct Color { ... };` 这行删掉，改为 `#include "types.h"`
- **`texture.h`**：include `types.h` 而不是 `framebuffer.h`
- **其他文件**：`framebuffer.h` 已经 include 了 `types.h`，间接获得 `Color`，无需改动

改完后依赖关系是：
```
types.h       → 无依赖（基础类型）
vec.h         → 无依赖（数学类型）
texture.h     → include types.h, vec.h
framebuffer.h → include types.h, vec.h, texture.h
model.h       → include vec.h
```
完全没有循环。

**3. 新建 `src/core/texture.h`**

```cpp
#pragma once

#include <string>
#include <vector>

#include "types.h"
#include "../math/vec.h"

class Texture {
public:
    Texture(const std::string& filename);   // 从 TGA 文件加载

    int getWidth()  const;
    int getHeight() const;

    // UV 采样：输入 [0,1] 范围的 UV，返回颜色
    Color sample(float u, float v) const;

private:
    int width, height;
    std::vector<Color> pixels;   // 一维存储，下标 = y * width + x
};
```

**3. 新建 `src/core/texture.cpp`，实现读取 TGA**

TGA 文件格式你在第一阶段已经用过了（写入），现在反过来**读取**：

读取流程：
```
1. 用 std::ifstream 以二进制模式打开文件
2. 读取 18 字节文件头
3. 从文件头解析宽度（字节 12-13，小端序 short）和高度（字节 14-15）
4. 根据文件头字节 2（图像类型）和字节 16（每像素位数）决定读取方式
   - 类型 2，24位：未压缩 BGR
   - 类型 2，32位：未压缩 BGRA（需要跳过 A 通道）
5. 逐像素读取 BGR，存入 pixels（注意转成 RGB）
```

**解析宽高（小端序）：**
```cpp
short w, h;
memcpy(&w, header + 12, 2);
memcpy(&h, header + 14, 2);
width  = w;
height = h;
```

**实现 `sample(float u, float v)` 函数：**

```
1. 把 UV 从 [0,1] 映射到像素坐标：
   int tx = (int)(u * width)    clamp 到 [0, width-1]
   int ty = (int)(v * height)   clamp 到 [0, height-1]

2. TGA 存储时 y=0 在底部，而我们的坐标系 y=0 也在底部，所以直接用 ty 即可

3. return pixels[ty * width + tx]
```

> **clamp 是什么：** 把一个值限制在某个范围内。`clamp(x, 0, max) = max(0, min(x, max))`。UV 插值后可能因为浮点误差略微超出 [0,1]，必须 clamp 防止越界。

**4. 更新 `CMakeLists.txt`**

把 `src/core/texture.cpp` 加入 `SOURCE` 列表。`types.h` 是纯头文件，不需要追加。

### 检查点
- [ ] `types.h` 创建完毕，`framebuffer.h` 和 `texture.h` 都改为 include `types.h`
- [ ] `Texture` 类编译通过
- [ ] 在 `main.cpp` 里临时加一行：`std::cout << texture.getWidth() << std::endl;`，能输出 1024
- [ ] 编译无

## 实验二：扩展 Model，解析 UV 坐标

### 目标
扩展 `Model` 类，在加载 `.obj` 时同时解析 `vt` 行（UV 坐标），并能根据面索引返回顶点的 UV。

### 背景知识：.obj 的 UV 索引

`.obj` 文件中，UV 坐标和顶点坐标是**分开存储**的，面定义里单独有 UV 索引：

```
vt 0.532 0.923    ← UV 坐标，独立编号
vt 0.231 0.451

f 1/1/1 2/2/2 3/3/3
  ^顶点  ^UV  ^法线
```

关键：**顶点索引和 UV 索引是独立的**，同一个顶点坐标可以对应不同的 UV（比如缝合边）。因此 `faces` 需要**同时存顶点索引和 UV 索引**。

### 步骤

**1. 修改 `model.h`**

新增私有成员：
```cpp
std::vector<Vec2> uvs;                      // 所有 UV 坐标
std::vector<std::vector<int>> faceUVs;      // 每个面的 UV 索引（与 faces 平行）
```

新增公有接口：
```cpp
Vec2 uv(int faceIdx, int vertIdx) const;    // 第 faceIdx 个面第 vertIdx 个顶点的 UV
```

**2. 修改 `model.cpp`**

**解析 `vt` 行**（在现有 `v` 行解析后面加）：
```cpp
if (type == "vt") {
    float u, v;
    iss >> u >> v;
    uvs.push_back(Vec2(u, v));
}
```

**修改 `f` 行解析**，同时提取 UV 索引：

当前代码用 `std::stoi(token)` 直接取整数，只取了顶点索引。现在需要同时取 UV 索引（`/` 后的第一个数字）：

```
token 可能的格式：
  "1"        → 只有顶点索引
  "1/2"      → 顶点/UV
  "1/2/3"    → 顶点/UV/法线
  "1//3"     → 顶点//法线（UV 缺失）
```

解析方法：用 `std::istringstream` 和 `getline` 以 `/` 为分隔符切分：

```cpp
std::istringstream tokenStream(token);
std::string part;
std::vector<std::string> parts;
while (std::getline(tokenStream, part, '/')) {
    parts.push_back(part);
}
int vertIdx = std::stoi(parts[0]) - 1;
int uvIdx   = (parts.size() > 1 && !parts[1].empty()) ? std::stoi(parts[1]) - 1 : -1;
```

把 `vertIdx` 存入 `faceVerts`，把 `uvIdx` 存入对应的 `faceUVIdxs`（临时 vector），最后 `faces.push_back(faceVerts)` 和 `faceUVs.push_back(faceUVIdxs)`。

**实现 `uv()` 函数：**
```cpp
Vec2 Model::uv(int faceIdx, int vertIdx) const {
    int uvIdx = faceUVs[faceIdx][vertIdx];
    if (uvIdx < 0) return Vec2(0.0f, 0.0f);   // 没有 UV 时返回原点
    return uvs[uvIdx];
}
```

### 检查点
- [ ] 编译通过
- [ ] 在 `main.cpp` 里打印 `model.uv(0, 0)`，能输出合理的 UV 坐标（值在 [0,1] 范围内）
- [ ] `model.uv(1, 2)` 和 `model.uv(0, 0)` 值不同（说明不同面的 UV 确实不一样）

## 实验三：升级 drawTriangle，支持 UV 插值

### 目标
在 `Framebuffer` 中新增一个支持纹理采样的 `drawTriangle` 重载，旧的两个版本保持不变。

### 步骤

**1. 在 `framebuffer.h` 中新增声明**

需要先 include texture.h（或者前向声明），然后添加：

```cpp
#include "texture.h"

// 新增：带 UV 插值和纹理采样的三角形
// v0/v1/v2：Vec3，xy 是屏幕坐标，z 是深度
// uv0/uv1/uv2：对应顶点的 UV 坐标
// tex：纹理对象引用
void drawTriangle(const Vec3& v0, const Vec3& v1, const Vec3& v2,
                  const Vec2& uv0, const Vec2& uv1, const Vec2& uv2,
                  const Texture& tex);
```

这样 `Framebuffer` 就有三个 `drawTriangle` 重载：
1. `drawTriangle(Vec2, Vec2, Vec2, Color)` —— 第一阶段，纯色2D
2. `drawTriangle(Vec3, Vec3, Vec3, Color)` —— 第三阶段，纯色3D+深度
3. `drawTriangle(Vec3, Vec3, Vec3, Vec2, Vec2, Vec2, Texture)` —— 第四阶段，纹理

**2. 在 `framebuffer.cpp` 中实现**

算法与第三阶段的 `drawTriangle(Vec3,Vec3,Vec3,Color)` 完全相同，只在"写入颜色"这一步改为从纹理采样：

```
伪代码：
for 每个像素 (x, y) in 包围盒：
    计算重心坐标 (α, β, γ)
    if 在三角形外，跳过
    
    插值深度：z = α*v0.z + β*v1.z + γ*v2.z
    深度测试：if z >= zbuffer[idx], 跳过
    更新深度缓冲
    
    【新增】插值 UV：
    float u = α*uv0.u + β*uv1.u + γ*uv2.u
    float v = α*uv0.v + β*uv1.v + γ*uv2.v
    
    【新增】纹理采样：
    Color color = tex.sample(u, v)
    
    setPixel(x, y, color)
```

> **Vec2 的 x/y vs u/v：** 你的 `Vec2` 成员叫 `x` 和 `y`，当它用作 UV 时，`x` 就是 `u`，`y` 就是 `v`。访问时写 `uv0.x`（u分量）和 `uv0.y`（v分量）即可。

### 检查点
- [ ] 新的 `drawTriangle` 重载编译通过
- [ ] 旧的两个 `drawTriangle` 仍然正常工作（编译通过，不需要重新验证功能）

## 实验五：理解透视矫正插值（重要概念，可选实践）

### 这是什么

你现在实现的 UV 插值是**线性插值**（直接用重心坐标加权平均），但在透视投影下，线性插值会产生错误——棋盘格纹理贴到有透视的平面时，格子边界会弯曲。

正确的做法是**透视矫正插值（Perspective-correct interpolation）**。

### 为什么线性插值在透视下是错的

透视投影把远处的物体压缩，近处放大。屏幕空间的等距步长，对应世界空间的**不等距**步长。如果在屏幕空间线性插值 UV，就相当于在世界空间做了不均匀插值，纹理会产生失真。

### 透视矫正插值的做法

在做透视除法之前，顶点的 UV 和深度是线性关系。利用这一点，把 UV 除以深度后再插值，最后乘回来：

```
每个顶点存储：(u/w, v/w, 1/w)，其中 w 是裁剪空间的 w 分量（即 clip.w）

插值：
  u_over_w = α*(u0/w0) + β*(u1/w1) + γ*(u2/w2)
  v_over_w = α*(v0/w0) + β*(v1/w1) + γ*(v2/w2)
  one_over_w = α*(1/w0) + β*(1/w1) + γ*(1/w2)

还原：
  u = u_over_w / one_over_w
  v = v_over_w / one_over_w
```

### 需要的改动

要实现透视矫正，需要：
1. `vertTransform` 同时返回裁剪空间的 `w` 分量（目前返回的 `Vec3` 中 z 是 NDC.z，需要额外传递 clip.w）
2. 在 `drawTriangle` 里用上面的公式替换线性插值

**这个实验是可选的。** african_head 模型平面度较高，线性插值和透视矫正差异不明显。如果你想亲眼看到差异，可以在场景里放一个正对摄像机的平面（两个三角形组成的矩形），贴上棋盘格纹理，分别用线性和透视矫正插值对比效果。

> **面试时能说什么：** "我知道在透视投影下直接线性插值 UV 会产生失真，正确做法是透视矫正插值，需要在裁剪空间除以 w 再插值然后还原。我的实现中做了/没做这个处理。"——能说清楚原理，不管有没有实现都是加分的。

## 常见问题

**Q：编译报错“Color 重复定义”？**  
A：确保 `Color` 只在 `types.h` 里定义一次。`framebuffer.h` 里原来的 `struct Color { ... };` 这行必须删掉，改为 `#include "types.h"`。如果两个地方都有定义就会冲突。

**Q：纹理贴出来是上下颠倒的？**  
A：TGA 文件的 y=0 在底部，你的坐标系 y=0 也在底部，理论上不应该颠倒。检查 `texture.cpp` 里读取像素时，是否从 `y=0` 开始按行读取（和写 TGA 时方向一致）。如果颠倒，在 `sample()` 里把 `ty` 改为 `height - 1 - ty` 试试。

**Q：纹理贴出来有明显的锯齿/像素感？**  
A：当前实现是**最近邻采样（Nearest Neighbor）**，即直接取最近的像素，所以会有像素感。平滑的方法是**双线性插值采样**，取周围4个像素加权平均。这是可以优化的点，但不是必须的。

**Q：纹理颜色偏暗或偏蓝？**  
A：检查 `texture.cpp` 读取时 BGR 到 RGB 的转换是否正确。TGA 存储是 BGR 顺序，读取时要把 `b` 存到 `Color.b`，`g` 存到 `Color.g`，`r` 存到 `Color.r`。

**Q：模型某些区域纹理坐标错误（有拉伸的条纹）？**  
A：检查 `model.cpp` 中 `f` 行的 UV 索引解析是否正确，确认索引减了 1。另外确认 `faceUVs` 和 `faces` 的大小是否相同（每个面都有对应的 UV 索引）。
