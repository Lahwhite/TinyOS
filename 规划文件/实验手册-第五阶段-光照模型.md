## 背景知识：光照是怎么算的

### 为什么需要法线

光照的核心思想是：**一个表面接收到的光照强度，取决于表面法线和光照方向的夹角。**

法线是垂直于表面的单位向量。夹角越小（法线越朝向光源），表面越亮；夹角超过90度（法线背对光源），表面在阴影中。

数学表达：`intensity = max(0, normal · lightDir)`

这就是**漫反射（Diffuse）**，是最基础的光照分量。

### Blinn-Phong 光照模型

完整的 Blinn-Phong 模型由三个分量组成：

```
最终颜色 = 环境光 + 漫反射 + 高光

环境光（Ambient）：
  color_ambient = ka * light_color
  作用：模拟场景中的间接光，防止背光面完全黑暗
  ka 是一个小系数，通常 0.1 左右

漫反射（Diffuse）：
  intensity = max(0, n · l)      n=法线，l=光照方向（单位向量）
  color_diffuse = kd * texture_color * intensity
  作用：根据面朝向决定明暗
  kd 通常为 1.0

高光（Specular）：
  h = normalize(l + v)           v=视线方向（顶点到摄像机，单位向量）
  spec = max(0, n · h) ^ shininess
  color_specular = ks * spec
  作用：产生高光亮斑，shininess 越大高光越集中
  ks 通常为 0.5~1.0，shininess 通常 32~128
```

> **为什么用半程向量 h 而不是反射向量？**  
> 原始 Phong 模型用反射向量，计算量更大。Blinn 改用光照方向和视线方向的半程向量 h，效果相近但计算更快，这就是 Blinn-Phong 的改进所在。

### 两种法线来源

**顶点法线（Vertex Normal）：**  
.obj 文件里存储的 `vn` 数据，每个顶点有一个法线，三角形内部通过插值得到逐像素法线。这是本阶段的基础实现。

**法线贴图（Normal Map）：**  
一张特殊的纹理图，每个像素存储了一个法线方向（用 RGB 编码 XYZ）。相比顶点法线，法线贴图能表达远超几何分辨率的表面细节，是现代渲染的标配。本阶段实验四实现。

## 实验一：扩展 Model，解析顶点法线

### 目标
扩展 `Model` 类，在加载 `.obj` 时同时解析 `vn` 行（顶点法线），并能根据面索引返回顶点法线。

### 背景知识
.obj 文件中，法线和顶点坐标同样是分开存储的：

```
vn 0.001 0.482 0.876    ← 法线向量（已归一化）
vn -0.234 0.123 0.965

f 1/1/1 2/2/2 3/3/3
  ^顶点  ^UV  ^法线索引
```

法线索引是 `f` 行每个顶点的第三个数字，与顶点索引、UV 索引独立编号。

### 步骤

**1. 修改 `model.h`**

新增私有成员：
```cpp
std::vector<Vec3> normals;               // 所有顶点法线
std::vector<std::vector<int>> faceNormals; // 每个面的法线索引（与 faces 平行）
```

新增公有接口：
```cpp
Vec3 normal(int faceIdx, int vertIdx) const;  // 第 faceIdx 个面第 vertIdx 个顶点的法线
```

**2. 修改 `model.cpp`**

**解析 `vn` 行**（在现有 `vt` 行解析旁边加）：
```cpp
} else if (type == "vn") {
    float x, y, z;
    iss >> x >> y >> z;
    normals.push_back(Vec3(x, y, z));
}
```

**修改 `f` 行解析**，在已有的顶点/UV 解析基础上同时取法线索引（第三个 `/` 分隔的部分）：

当前 `parts` 已经按 `/` 切分好了，只需取第三个元素：
```cpp
int normalIdx = (parts.size() > 2 && !parts[2].empty()) ? std::stoi(parts[2]) - 1 : -1;
```

把 `normalIdx` 存入 `faceNormalIdxs`（临时 vector），最后 `faceNormals.push_back(faceNormalIdxs)`。

**实现 `normal()` 函数：**
```cpp
Vec3 Model::normal(int faceIdx, int vertIdx) const {
    int idx = faceNormals[faceIdx][vertIdx];
    if (idx < 0) return Vec3(0.0f, 0.0f, 1.0f);  // 没有法线时默认朝前
    return normals[idx];
}
```

## 实验二：实现面法线着色（热身）

### 目标
在渲染循环中，先用**面法线**（不是顶点法线插值）做一个最简单的漫反射着色，验证光照计算逻辑是否正确。

这是第三阶段实验四的"正式版"——那时用的是固定颜色乘以强度，现在结合纹理颜色。

### 步骤

**修改 `main.cpp` 的渲染循环**

在循环里，取顶点的**世界坐标**计算面法线，然后算漫反射强度：

```cpp
Vec3 lightDir = Vec3::normalize(Vec3(0.0f, 0.0f, 1.0f));  // 光从 +z 方向来

for (int i = 0; i < model.numFaces(); ++i) {
    Vec3 wv0 = model.vert(i, 0);   // 世界坐标顶点
    Vec3 wv1 = model.vert(i, 1);
    Vec3 wv2 = model.vert(i, 2);

    // 面法线（用世界坐标计算）
    Vec3 faceNormal = Vec3::normalize((wv1 - wv0).cross(wv2 - wv0));

    // 漫反射强度
    float intensity = faceNormal.dot(lightDir);
    if (intensity <= 0.0f) continue;   // 背面剔除

    // 屏幕坐标变换
    Vec3 p0 = vertTransform(wv0, mvp, 512, 512);
    Vec3 p1 = vertTransform(wv1, mvp, 512, 512);
    Vec3 p2 = vertTransform(wv2, mvp, 512, 512);

    Vec2 uv0 = model.uv(i, 0);
    Vec2 uv1 = model.uv(i, 1);
    Vec2 uv2 = model.uv(i, 2);

    // 把光照强度乘到纹理颜色上
    // 注意：纹理颜色 * intensity，需要在 drawTriangle 外面处理，
    // 或者传入 intensity 让 drawTriangle 内部乘
    // 暂时用固定颜色乘以强度测试：
    Color color{
        (uint8_t)(255 * intensity),
        (uint8_t)(255 * intensity),
        (uint8_t)(255 * intensity)
    };
    framebuffer.drawTriangle(p0, p1, p2, color);  // 用第三阶段的纯色版本
}
```

> **为什么先不用纹理？**  
> 把光照和纹理分开验证，先确认光照计算正确（人头有明暗变化），再加纹理。排错更容易。

### 检查点
- [ ] 人头有明暗变化（不是纯白色）
- [ ] 背面（脑袋后面）被剔除，没有渲染
- [ ] 改变 `lightDir` 方向，明暗随之变化
- [ ] 侧光（`lightDir = Vec3(1,0,0)`）时，左右两侧明暗差异明显

## 实验三：顶点法线插值，实现逐像素光照

### 目标
把面法线换成**顶点法线插值**，实现逐像素的 Blinn-Phong 光照，加上纹理，得到有真实感的渲染结果。

### 背景知识：为什么要插值法线

面法线是整个三角形共用一个法线，导致三角形边界上有明显的明暗跳跃（也叫 Mach band 效应）。

顶点法线插值（Phong Shading）：每个顶点有自己的法线，三角形内部每个像素的法线通过重心坐标插值得到，明暗过渡平滑。

### 步骤

**1. 在 `framebuffer.h` 中新增第四个 `drawTriangle` 重载**

```cpp
// 带法线插值的 Blinn-Phong 着色
void drawTriangle(
    const Vec3& p0, const Vec3& p1, const Vec3& p2,    // 屏幕坐标(xy)+深度(z)
    const Vec2& uv0, const Vec2& uv1, const Vec2& uv2, // UV 坐标
    const Vec3& n0,  const Vec3& n1,  const Vec3& n2,  // 顶点法线（世界坐标）
    const Texture& diffuseTex,                          // 漫反射纹理
    const Vec3& lightDir,                               // 光照方向（单位向量，世界坐标）
    const Vec3& viewDir                                 // 视线方向（单位向量，世界坐标）
);
```

**2. 在 `framebuffer.cpp` 中实现**

算法流程（在已有深度测试基础上扩展）：

```
for 每个像素 (x, y) in 包围盒（记得裁剪到屏幕范围）：
    计算重心坐标 (α, β, γ)
    if 在三角形外，跳过

    深度测试（同前）

    【插值法线】
    Vec3 n = α*n0 + β*n1 + γ*n2
    n = Vec3::normalize(n)    ← 插值后长度不再是1，需要重新归一化

    【插值 UV，采样纹理】
    float u = α*uv0.x + β*uv1.x + γ*uv2.x
    float v = α*uv0.y + β*uv1.y + γ*uv2.y
    Color texColor = diffuseTex.sample(u, v)

    【Blinn-Phong 光照计算】
    // 漫反射
    float diff = max(0.0f, n.dot(lightDir))

    // 高光（半程向量）
    Vec3 h = Vec3::normalize(lightDir + viewDir)
    float spec = pow(max(0.0f, n.dot(h)), 32.0f)   // shininess = 32

    // 环境光系数
    float ambient = 0.1f

    // 合并强度
    float r = std::min(1.0f, texColor.r / 255.0f * (ambient + diff) + spec * 0.5f)
    float g = std::min(1.0f, texColor.g / 255.0f * (ambient + diff) + spec * 0.5f)
    float b = std::min(1.0f, texColor.b / 255.0f * (ambient + diff) + spec * 0.5f)

    Color finalColor{(uint8_t)(r*255), (uint8_t)(g*255), (uint8_t)(b*255)}
    setPixel(x, y, finalColor)
```

> **注意浮点精度：** `texColor.r` 是 `uint8_t`（0-255），参与光照计算时先除以 255 转成 [0,1] 浮点，计算完再乘回 255 转成 uint8_t。最后用 `std::min(1.0f, ...)` 防止超过1.0导致溢出。

**3. 修改 `main.cpp` 渲染循环**

```cpp
Vec3 lightDir = Vec3::normalize(Vec3(1.0f, 1.0f, 1.0f));   // 光从右上前方
Vec3 eye      = Vec3(0.0f, 0.0f, 3.0f);                     // 摄像机位置（与 lookAt 一致）

for (int i = 0; i < model.numFaces(); ++i) {
    Vec3 wv0 = model.vert(i, 0);
    Vec3 wv1 = model.vert(i, 1);
    Vec3 wv2 = model.vert(i, 2);

    Vec3 p0 = vertTransform(wv0, mvp, 512, 512);
    Vec3 p1 = vertTransform(wv1, mvp, 512, 512);
    Vec3 p2 = vertTransform(wv2, mvp, 512, 512);

    Vec2 uv0 = model.uv(i, 0);
    Vec2 uv1 = model.uv(i, 1);
    Vec2 uv2 = model.uv(i, 2);

    Vec3 n0 = model.normal(i, 0);   // 顶点法线
    Vec3 n1 = model.normal(i, 1);
    Vec3 n2 = model.normal(i, 2);

    // 视线方向：从模型中心指向摄像机（近似用 eye 方向）
    Vec3 viewDir = Vec3::normalize(eye);

    framebuffer.drawTriangle(p0, p1, p2, uv0, uv1, uv2, n0, n1, n2,
                             texture, lightDir, viewDir);
}
```

- [ ] 能看到高光亮斑（脸部最亮的区域）
- [ ] 背光面有环境光（不是纯黑色，而是暗色）

## 实验四：法线贴图（Normal Map）

### 目标
用法线贴图替代顶点法线，大幅提升表面细节，实现皮肤纹理、毛孔等细节的光照效果。

### 背景知识：法线贴图的编码

法线贴图是一张 RGB 纹理，每个像素存储了一个法线向量：
- R 通道 → 法线 X 分量
- G 通道 → 法线 Y 分量  
- B 通道 → 法线 Z 分量

存储时把 [-1, 1] 范围映射到 [0, 255]：

```
存储：pixel_value = (normal + 1.0) / 2.0 * 255
读取：normal = pixel_value / 255.0 * 2.0 - 1.0
```

法线贴图通常偏蓝色（因为大多数法线 Z 分量接近 1，即朝外）。

### 两种空间的法线

法线贴图存储的法线有两种空间：

**切线空间（Tangent Space）法线贴图（更常见）：**  
法线相对于表面局部坐标系定义。优点是可以被不同模型复用。需要在渲染时把光照方向变换到切线空间，或把切线空间法线变换到世界空间。

**世界/对象空间法线贴图：**  
法线直接是世界坐标，读出来就能用，实现简单。african_head 配套的法线贴图是对象空间的。

### 步骤

**1. 下载法线贴图**

从 tinyrenderer 的仓库下载 `african_head_nm.tga`，放到 `game/assets/` 目录下：
```
https://github.com/ssloy/tinyrenderer/tree/master/obj/african_head
```

**2. 在 `main.cpp` 中加载法线贴图**

```cpp
Texture normalMap("../assets/african_head_nm.tga");
```

**3. 在渲染循环中从法线贴图取法线**

把顶点法线的插值替换为从法线贴图采样：

```cpp
// 先插值 UV，然后用 UV 从法线贴图采样法线
// （在 drawTriangle 内部，或在外面计算后传入）
```

最简单的做法是在 `framebuffer.cpp` 的光照 `drawTriangle` 里新增一个参数 `normalMap`：

```cpp
// 在计算法线时，用法线贴图的采样替代顶点插值法线：
Color nmSample = normalMap.sample(u, v);
Vec3 n(
    nmSample.r / 255.0f * 2.0f - 1.0f,   // R → X
    nmSample.g / 255.0f * 2.0f - 1.0f,   // G → Y
    nmSample.b / 255.0f * 2.0f - 1.0f    // B → Z
);
n = Vec3::normalize(n);
```

然后把这个 `n` 代入 Blinn-Phong 公式，替换掉顶点插值的法线。

**4. 对比两个版本的效果**

运行两个版本，对比输出图片：
- 顶点法线：法线分辨率低，表面平滑但缺乏细节
- 法线贴图：每个像素都有独立法线，皮肤纹理、细节清晰

## 常见问题

**Q：人头整体偏黑，几乎看不到纹理颜色？**  
A：检查 `ambient` 系数是否太小，以及 `diff` 的计算是否正确。先把光照方向设置成 `Vec3(0,0,1)`（正面打光），确认正面是亮的。如果正面也黑，检查法线的方向是否正确（`dot` 结果应该是正数）。

**Q：高光区域颜色溢出变成纯白色大片？**  
A：检查是否做了 `std::min(1.0f, ...)` 的 clamp。高光系数（`spec * 0.5f`）加上漫反射很容易超过1.0，必须 clamp 到 [0, 1]。也可以降低高光系数 `ks`。

**Q：法线插值后法线方向错了（整体偏暗或偏亮）？**  
A：检查顶点法线是否已经归一化（.obj 里的 `vn` 通常已经归一化，但插值后必须再次归一化）。另外确认法线方向：african_head 模型法线应该朝外（朝向 +z 方向大致），如果整体偏暗，试试 `-lightDir` 看是否变亮，说明法线方向反了。

**Q：法线贴图读出来颜色不对（全是蓝色）？**  
A：法线贴图偏蓝是正常的（大多数法线 Z 分量接近 1）。但如果读出来的 n.z 不接近 1，可能是 RGB 通道顺序读错了，检查 `texture.cpp` 里读取是否正确地把 B 通道的值赋给了 `Color.b`。

**Q：`std::pow` 的 `shininess` 调多少合适？**  
A：shininess 越大，高光越集中越亮（金属感）；越小，高光越散越暗（磨砂感）。african_head 皮肤通常用 32-64，觉得效果不对可以在 16~128 之间调整。

**Q：视线方向 `viewDir` 怎么算更准确？**  
A：本阶段用摄像机位置的方向（`normalize(eye)`）近似。更准确的做法是用每个像素对应的世界坐标来计算视线方向，但这需要反投影，实现更复杂。对于这个 demo，摄像机方向的近似已经足够得到好效果。
