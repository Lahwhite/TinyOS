# 实验手册 · TinyOS 第二阶段：中断，让 OS 能响应事件

> **阶段目标：** 时钟每隔约 10ms 触发一次中断，每秒在屏幕上打印一个数字；键盘按键后打印对应字符。  
> **预计时间：** 1～2 周，每天约 1 小时  
> **前置条件：** 第一阶段完成，能在 QEMU 上打印字符串  
> **本阶段新增文件：**
> ```
> TinyOS/
> └── kernel/
>     ├── idt.h/c         ← 中断描述符表
>     ├── isr.asm         ← 中断服务例程（汇编部分）
>     ├── isr.h/c         ← 中断处理函数（C部分）
>     ├── pic.h/c         ← 可编程中断控制器
>     └── timer.h/c       ← 时钟中断
> ```

## 实验一：理解中断机制

### 背景知识：中断是 OS 的心跳

MiniOS 里，你的调度器是这样工作的：

```cpp
scheduler.run("RR", mem);   // 手动调用，顺序执行
```

真实 OS 里，没有人"手动"调度进程。CPU 靠**中断**驱动一切：

```
CPU 正在执行进程 A 的代码
    ↓
时钟芯片发出一个电信号（每隔约 10ms）
    ↓
CPU 暂停当前指令，保存所有寄存器状态（上下文）
    ↓
跳转到内核的中断处理函数（Interrupt Service Routine）
    ↓
内核决定：继续运行 A，还是切换到 B？
    ↓
恢复某个进程的上下文，继续执行
```

**中断分类：**

| 类型 | 来源 | 例子 |
|------|------|------|
| 硬件中断（IRQ）| 外部硬件设备 | 时钟、键盘、网卡、磁盘完成 |
| 软件中断 | `int` 汇编指令 | 系统调用（`int 0x80`） |
| 异常 | CPU 检测到错误 | 除以零、缺页、段错误 |

x86 有 256 个中断号（0～255）：
- 0～31：CPU 异常（固定含义，如 0 = 除以零，14 = 缺页）
- 32～47：硬件中断 IRQ（时钟 = IRQ0 = 中断号32，键盘 = IRQ1 = 33）
- 48～255：软件中断（OS 自定义，如系统调用用 0x80）

**IDT（中断描述符表）：**

IDT 是一个数组，每个元素（门描述符）记录：当某个中断号触发时，应该跳转到哪个函数地址。类比 MiniOS 的调度算法，IDT 就是"中断号 → 处理函数"的映射表。

## 实验二：初始化 IDT，注册默认处理函数

### 目标
建立 IDT，为所有 256 个中断注册一个默认处理函数（暂时只是打印错误信息）。

### 步骤

**1. 新建 `kernel/idt.h`**

```c
#ifndef IDT_H
#define IDT_H

#include <stdint.h>    // 裸机环境可用 stdint.h（GCC 内置，不依赖 OS）

// IDT 中一个门描述符（8字节）
struct idt_entry {
    uint16_t offset_low;    // 处理函数地址低16位
    uint16_t selector;      // 代码段选择子（固定用 0x08）
    uint8_t  zero;          // 必须为0
    uint8_t  type_attr;     // 类型和属性（0x8E = 32位中断门，Ring 0）
    uint16_t offset_high;   // 处理函数地址高16位
} __attribute__((packed));  // 禁止编译器填充对齐字节

// IDTR 寄存器格式（用于 lidt 指令）
struct idt_ptr {
    uint16_t limit;    // IDT 大小（字节数-1）
    uint32_t base;     // IDT 物理地址
} __attribute__((packed));

void idt_init();
void idt_set_gate(int num, uint32_t handler_addr);

#endif
```

**2. 新建 `kernel/idt.c`**

实现 `idt_set_gate`：
```
输入：中断号 num，处理函数地址 handler_addr
操作：
  idt[num].offset_low  = handler_addr & 0xFFFF
  idt[num].selector    = 0x08            （内核代码段）
  idt[num].zero        = 0
  idt[num].type_attr   = 0x8E            （32位中断门，Ring 0）
  idt[num].offset_high = (handler_addr >> 16) & 0xFFFF
```

实现 `idt_init`：
```
for i in 0..255:
    idt_set_gate(i, default_handler 的地址)

设置 idt_ptr.limit = 256 * 8 - 1
设置 idt_ptr.base  = IDT 数组地址

执行 lidt 指令（用汇编内联：asm("lidt %0" : : "m"(idt_ptr))）
```

**3. 新建 `kernel/isr.asm`，实现汇编层中断入口**

每个中断的入口必须用汇编写，原因：C 不能直接处理中断的寄存器保存/恢复。

```nasm
[bits 32]
[extern isr_handler]    ; C 语言的通用处理函数

; 宏：生成一个中断入口（保存寄存器后调用 C 函数）
%macro ISR_NOERRCODE 1
global isr%1
isr%1:
    push dword 0        ; 压入一个假的错误码（无错误码的中断需要占位）
    push dword %1       ; 压入中断号
    jmp isr_common
%endmacro

%macro ISR_ERRCODE 1
global isr%1
isr%1:
    push dword %1       ; 这类中断 CPU 自动压入了错误码，直接压中断号
    jmp isr_common
%endmacro

; CPU 异常（0～31）
ISR_NOERRCODE 0    ; 除以零
ISR_NOERRCODE 1    ; 调试
ISR_NOERRCODE 2    ; NMI
ISR_NOERRCODE 3    ; 断点
; ... 共32个，部分有错误码
ISR_ERRCODE  8     ; 双重错误（有错误码）
ISR_ERRCODE  14    ; 缺页（有错误码）
; ...（其余省略，参考 OSDev Wiki 哪些有错误码）

; 通用中断处理入口
isr_common:
    pusha               ; 保存所有通用寄存器（EAX/ECX/EDX/EBX/ESP/EBP/ESI/EDI）
    push ds
    push es
    push fs
    push gs

    mov ax, 0x10        ; 切换到内核数据段
    mov ds, ax
    mov es, ax

    call isr_handler    ; 调用 C 处理函数（栈上已有所有寄存器）

    pop gs
    pop fs
    pop es
    pop ds
    popa                ; 恢复所有通用寄存器
    add esp, 8          ; 弹出中断号和错误码
    iret                ; 中断返回（恢复 CS/EIP/EFLAGS）
```

**4. 新建 `kernel/isr.h/c`，实现 C 层处理函数**

```c
// 传给 C 处理函数的寄存器快照结构
struct registers {
    uint32_t gs, fs, es, ds;           // 段寄存器
    uint32_t edi, esi, ebp, esp;       // pusha 压入的
    uint32_t ebx, edx, ecx, eax;
    uint32_t int_num, err_code;        // isr.asm 压入的
    uint32_t eip, cs, eflags;          // CPU 自动压入的
    uint32_t user_esp, ss;             // Ring 切换时压入（用户态才有）
};

void isr_handler(struct registers regs) {
    // 暂时只打印中断号
    vga_puts("Exception: ");
    // 打印 regs.int_num（需要实现 kprintf 或 int转字符串函数）
}
```

**5. 在 `kernel_main` 里调用 `idt_init()`，然后用 `sti` 开启中断**

```c
void kernel_main() {
    vga_init();
    vga_clear();
    vga_puts("Initializing IDT...\n");
    idt_init();
    asm volatile("sti");    // 开启硬件中断
    vga_puts("Interrupts enabled.\n");
    while (1) {}
}
```

### 检查点
- [ ] IDT 256 个条目全部设置，编译通过
- [ ] `asm("lidt ...")` 成功加载 IDT（QEMU 不崩溃）
- [ ] 主动触发一个异常（如 `int 0`），能看到处理函数被调用

## 实验三：初始化 PIC，接收硬件中断

### 目标
初始化 8259A 可编程中断控制器，让时钟（IRQ0）和键盘（IRQ1）的信号能到达 CPU。

### 背景知识：为什么需要 PIC？

x86 CPU 只有一个 `INTR` 引脚接收中断信号。而系统里有时钟、键盘、磁盘等多个设备，都需要发中断。**8259A PIC**（Programmable Interrupt Controller）是中间的"多路复用器"，把多个设备的信号合并后送给 CPU，并告诉 CPU 是哪个设备发的。

**实模式下，IRQ0-IRQ7 默认映射到中断号 8-15，和 CPU 异常冲突！** 进入保护模式后必须重新映射：

```
IRQ0（时钟）  → 中断号 32
IRQ1（键盘）  → 中断号 33
IRQ2（级联）  → 中断号 34
...
IRQ7          → 中断号 39

IRQ8~IRQ15   → 中断号 40~47（第二片 PIC）
```

**端口操作：** PIC 通过 I/O 端口控制，用 `outb`（out byte）发命令：

| 端口 | 作用 |
|------|------|
| `0x20` | 主 PIC 命令端口 |
| `0x21` | 主 PIC 数据端口（中断屏蔽） |
| `0xA0` | 从 PIC 命令端口 |
| `0xA1` | 从 PIC 数据端口 |

### 步骤

**1. 新建 `kernel/pic.h`**

```c
#ifndef PIC_H
#define PIC_H

void pic_init();
void pic_send_eoi(int irq);   // 发送"中断处理完毕"信号（End of Interrupt）

// 端口 I/O 函数（汇编内联实现）
static inline void outb(uint16_t port, uint8_t val) {
    asm volatile("outb %0, %1" : : "a"(val), "Nd"(port));
}

static inline uint8_t inb(uint16_t port) {
    uint8_t ret;
    asm volatile("inb %1, %0" : "=a"(ret) : "Nd"(port));
    return ret;
}

#endif
```

**2. 新建 `kernel/pic.c`，实现 PIC 初始化**

```c
void pic_init() {
    // 初始化序列（ICW1-ICW4，固定套路）
    outb(0x20, 0x11);   // 主 PIC：开始初始化，边沿触发
    outb(0xA0, 0x11);   // 从 PIC：开始初始化

    outb(0x21, 32);     // 主 PIC IRQ0 映射到中断号 32
    outb(0xA1, 40);     // 从 PIC IRQ8 映射到中断号 40

    outb(0x21, 4);      // 主 PIC：IRQ2 接从 PIC（级联）
    outb(0xA1, 2);      // 从 PIC：级联标识

    outb(0x21, 0x01);   // 主 PIC：8086 模式
    outb(0xA1, 0x01);   // 从 PIC：8086 模式

    // 屏蔽所有中断（除了 IRQ0 时钟和 IRQ1 键盘）
    // 位为 0 = 开启，位为 1 = 屏蔽
    outb(0x21, 0b11111100);   // 开启 IRQ0、IRQ1
    outb(0xA1, 0xFF);          // 屏蔽所有从 PIC 中断
}

void pic_send_eoi(int irq) {
    if (irq >= 8) outb(0xA0, 0x20);   // 如果是从 PIC 的 IRQ，要同时发给从 PIC
    outb(0x20, 0x20);                   // 发给主 PIC
}
```

**3. 为 IRQ 注册中断入口（在 isr.asm 里追加）**

```nasm
; 硬件中断（IRQ0～IRQ15 对应中断号 32～47）
%macro IRQ 2
global irq%1
irq%1:
    push dword 0
    push dword %2    ; 中断号
    jmp irq_common
%endmacro

IRQ 0, 32    ; 时钟
IRQ 1, 33    ; 键盘
; ... 其余 IRQ

[extern irq_handler]
irq_common:
    pusha
    ; ... 同 isr_common
    call irq_handler
    ; ... 恢复寄存器
    iret
```

**4. 在 `kernel_main` 里调用 `pic_init()`**

```c
idt_init();
pic_init();
asm volatile("sti");
```

### 检查点
- [ ] `pic_init()` 调用后 QEMU 不崩溃
- [ ] 能解释 `pic_send_eoi` 的作用（中断处理完必须发 EOI，否则 PIC 不会再发同类中断）

## 实验四：时钟中断，让时间流动

### 目标
实现时钟中断处理函数，每次时钟中断将计数器加 1，每隔约 100 次打印一次（约 1 秒）。

### 背景知识：PIT（可编程间隔定时器）

x86 有一个 **8253/8254 PIT**（Programmable Interval Timer），连接到 IRQ0。它按固定频率发中断：**默认 ~18.2Hz**（每秒约 18 次中断）。

可以重新配置 PIT 的频率。设置 1000Hz（每毫秒一次）的方法：

```
除数 = 1193180 / 目标频率
outb(0x43, 0x36)           // 命令：通道0，模式3，二进制
outb(0x40, 除数 & 0xFF)    // 写入除数低8位
outb(0x40, 除数 >> 8)      // 写入除数高8位
```

### 步骤

**1. 新建 `kernel/timer.h/c`**

```c
// timer.h
void timer_init(uint32_t frequency);   // 设置时钟频率（Hz）
uint32_t timer_get_ticks();            // 返回启动以来的时钟次数

// timer.c
static uint32_t ticks = 0;

void timer_callback(struct registers regs) {
    ticks++;
    if (ticks % 100 == 0) {
        vga_puts("Tick: ");
        // 打印 ticks / 100（需要实现整数转字符串）
    }
    pic_send_eoi(0);   // 必须发 EOI，否则时钟中断不再触发
}

void timer_init(uint32_t frequency) {
    // 注册时钟中断处理函数到 IRQ0（中断号32）
    idt_set_gate(32, (uint32_t)timer_callback);
    
    // 设置 PIT 频率
    uint32_t divisor = 1193180 / frequency;
    outb(0x43, 0x36);
    outb(0x40, divisor & 0xFF);
    outb(0x40, divisor >> 8);
}
```

**2. 实现整数转字符串（`lib/string.c` 追加）**

```c
void itoa(int n, char* buf) {
    // 将整数 n 转换为十进制字符串，存入 buf
    // 处理负数、0、正数三种情况
}
```

**3. 在 `kernel_main` 里初始化时钟**

```c
timer_init(100);   // 100Hz，每10ms一次中断
asm volatile("sti");
while (1) { asm volatile("hlt"); }  // hlt：等待中断，节省 CPU
```

### 检查点
- [ ] 屏幕上每秒打印一次 `Tick: 1`、`Tick: 2`...
- [ ] 理解为什么中断处理函数必须调用 `pic_send_eoi`
- [ ] 理解 `hlt` 指令的作用（让 CPU 休眠，等待中断，避免空转）

## 实验五：键盘中断，接收用户输入

### 目标
实现键盘中断处理函数，按下键盘后在屏幕上打印对应字符。

### 背景知识：键盘扫描码

键盘按键时，键盘控制器通过 IRQ1 发中断，并将**扫描码**（Scan Code）放到 I/O 端口 `0x60`。扫描码不是 ASCII 码，需要查表转换：

```
0x1E → 'a'
0x30 → 'b'
0x2E → 'c'
...
```

键盘按下发一个扫描码，松开发一个高位为 1 的扫描码（按下码 | 0x80）。

### 步骤

**1. 新建 `drivers/keyboard.h/c`**

需要实现：
- 一个扫描码到 ASCII 的映射表（数组，第 i 个元素是扫描码 i 对应的字符）
- `keyboard_init()`：把键盘处理函数注册到 IDT 第 33 项（IRQ1）
- `keyboard_handler(struct registers regs)`：
  ```
  从端口 0x60 读一个字节（扫描码）
  如果最高位为 1：键被松开，忽略
  否则：查表得到字符，调用 vga_putchar 打印
  pic_send_eoi(1)
  ```

**2. 在 `kernel_main` 里调用 `keyboard_init()`**

```c
idt_init();
pic_init();
timer_init(100);
keyboard_init();
asm volatile("sti");
```

### 检查点
- [ ] 按 `a`、`b`、`c` 等键，屏幕上出现对应字符
- [ ] 按 Enter 键，光标换到下一行
- [ ] 键盘和时钟中断互不干扰，同时工作

## 常见问题

**Q：开启中断后 QEMU 立刻重启/三重错误？**  
A：说明某个中断触发了但 IDT 里没有有效的处理函数。检查：
1. IDT 是否正确加载（`lidt` 指令）
2. 所有 256 个 IDT 条目是否都指向了合法的处理函数地址
3. PIC 是否正确初始化（默认 IRQ 频繁触发）

**Q：时钟中断不触发（屏幕没有 Tick）？**  
A：按顺序检查：
1. `pic_init()` 是否在 `sti` 之前调用
2. IRQ0 是否被屏蔽（`outb(0x21, 0b11111100)` 应该开启 IRQ0）
3. 时钟中断的 IDT 条目（32号）是否指向了 `timer_callback`
4. `timer_callback` 里是否调用了 `pic_send_eoi(0)`

**Q：键盘按键没反应？**  
A：先确认时钟中断正常（证明 PIC 工作）。然后检查 IRQ1 是否未被屏蔽，以及 IDT 33 号条目是否正确注册。
