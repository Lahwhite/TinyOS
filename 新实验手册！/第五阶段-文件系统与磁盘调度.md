## 实验一：理解 inode，定义文件系统数据结构

### 目标
定义 `Inode` 结构体和 `FileSystem` 类的骨架，初始化后打印出根目录。

### 背景知识：文件名在哪里？

很多人以为文件名存在文件本身里，实际上不是。UNIX 文件系统的设计是：

```
文件名 → inode 号 → inode → 数据块
```

- **inode（索引节点）：** 存储文件的所有元数据（大小、权限、数据块指针），**不包含文件名**
- **目录：** 也是一种文件，它的内容是 `文件名 → inode号` 的映射表
- **文件名：** 存在父目录的映射表里，不在 inode 里

为什么这样设计？因为同一个 inode 可以有多个名字（硬链接）——不同目录里两个不同名字指向同一个 inode，文件只有一份数据。

**inode 包含什么：**

| 字段 | 类型 | 含义 |
|------|------|------|
| `inode_num` | `int` | inode 编号（唯一） |
| `is_directory` | `bool` | true = 目录，false = 普通文件 |
| `size` | `int` | 文件大小（字节） |
| `ref_count` | `int` | 引用计数（有几个硬链接指向此 inode） |
| `permissions` | `std::string` | 权限字符串，如 `"rw-r--r--"` |
| `content` | `std::string` | 文件内容（简化：直接存字符串，不模拟磁盘块） |
| `entries` | `std::map<string,int>` | 仅目录使用：文件名 → inode号 |

### 步骤

**1. 新建 `src/filesystem/filesystem.h`**

```
struct Inode {
    // 上表中所有字段
};

class FileSystem {
public:
    FileSystem();           // 初始化：创建根目录 inode（编号0）

    void printTree() const; // 以树形打印目录结构

private:
    std::vector<Inode> inodes;     // inode 表（下标即 inode 号）
    std::vector<bool>  inode_used; // 每个 inode 是否在使用
    int next_inode_num;            // 下一个可分配的 inode 号

    // 工具函数：分配一个新 inode，返回其编号
    int allocateInode();
};
```

**2. 新建 `src/filesystem/filesystem.cpp`**

实现构造函数：
```
inode 表初始化（预留 64 个位置）
创建根目录 inode（编号0）：
  is_directory = true
  ref_count = 2
  permissions = "rwxr-xr-x"
  entries["."]  = 0   （"." 指向自身）
  entries[".."] = 0   （根目录的 ".." 也指向自身）
```

实现 `allocateInode`：扫描 `inode_used`，找第一个未使用的，标记并返回编号。

实现 `printTree`：递归打印目录结构（类似 `tree` 命令的效果）。

**3. 在 `main.cpp` 中创建 FileSystem 并打印初始状态**

```cpp

fs.printTree();
```

/                    ← 根目录（inode 0）
```

## 实验二：实现 mkdir 和 touch，建立目录树

### 目标
给 `FileSystem` 添加 `mkdir` 和 `touch` 函数，能在当前目录下创建子目录和空文件。

### 背景知识：路径解析

为了简单起见，本阶段的文件系统**不做完整路径解析**（不支持 `/home/user/file.txt` 这样的绝对路径）。

用一个"当前目录"的 inode 号来模拟 shell 里的 `cd`：

```
int current_dir = 0;   // 初始在根目录
```

`mkdir "home"` 的含义：在 current_dir 对应的目录里创建一个名为 "home" 的子目录。

### 步骤

**1. 在 `filesystem.h` 中新增函数声明**

```cpp
// 在当前目录下创建子目录，返回新 inode 号（失败返回 -1）
int mkdir(const std::string& name);

// 在当前目录下创建空文件，返回新 inode 号（失败返回 -1）
int touch(const std::string& name);

// 进入子目录（cd），返回 false 如果目录不存在
bool cd(const std::string& name);

// 返回上级目录（cd ..）
void cdUp();

// 列出当前目录内容（ls）
void ls() const;

// 返回当前路径字符串（用于提示符显示）
std::string pwd() const;

// 内部维护：当前目录 inode 号 和 路径栈
int current_dir_inode;
std::vector<std::pair<std::string, int>> path_stack;  // (目录名, inode号)
```

**2. 实现 `mkdir`**

```
检查当前目录中是否已有同名条目（有则返回 -1）

分配新 inode num
初始化新目录 inode：
  is_directory = true
  ref_count = 2
  entries["."]  = num
  entries[".."] = current_dir_inode

在当前目录的 entries 里添加：name → num
当前目录的 ref_count++（因为新目录的 ".." 指向当前目录）

打印：[FS] mkdir: /当前路径/name (inode num)
返回 num
```

**3. 实现 `touch`**

类似 `mkdir`，区别：
- `is_directory = false`
- `ref_count = 1`
- 不需要 "." 和 ".." 条目

**4. 实现 `cd` 和 `cdUp`**

```
cd(name):
  检查当前目录 entries 里有没有 name
  检查对应 inode 是不是目录（is_directory == true）
  更新 current_dir_inode = 对应 inode 号
  将 (name, inode号) 压入 path_stack

cdUp():
  如果 path_stack 为空（已在根目录），不操作
  否则：弹出 path_stack 栈顶，current_dir_inode = 新栈顶的 inode 号
```

**5. 实现 `ls`**

```
打印当前目录 entries 里的所有条目（跳过 "." 和 ".."）
格式：inode号  类型  名称
```

**6. 在 `main.cpp` 中演示**

```cpp
fs.mkdir("home");
fs.mkdir("etc");
fs.cd("home");
fs.mkdir("user");
fs.touch("readme.txt");
fs.cdUp();
fs.printTree();
```

### 预期输出

```
=== 目录操作 ===
[FS] mkdir: /home (inode 1)
[FS] mkdir: /etc  (inode 2)
[FS] cd: /home
[FS] mkdir: /home/user (inode 3)
[FS] touch: /home/readme.txt (inode 4)

=== 目录树 ===
/
├── home/
│   ├── user/
│   └── readme.txt
└── etc/
```

### 检查点
- [ ] `mkdir` 正确创建目录 inode，并在父目录的 entries 里注册
- [ ] `touch` 正确创建文件 inode
- [ ] `cd` 和 `cdUp` 能正确切换当前目录
- [ ] `printTree` 能递归显示完整目录结构

## 实验三：实现 write 和 read，进程读写文件

### 目标
给文件系统添加读写功能，然后让 PCB 能触发文件操作——进程"运行"时写日志文件。

### 步骤

**1. 在 `filesystem.h` 中新增 write/read 函数**

```cpp
// 向当前目录下名为 name 的文件写入 content（覆盖式写入）
// 成功返回写入字节数，失败返回 -1
int write(const std::string& name, const std::string& content);

// 读取当前目录下名为 name 的文件内容
// 成功返回内容字符串，失败返回空字符串
std::string read(const std::string& name);

// 查看文件的 inode 信息（stat）
void stat(const std::string& name) const;
```

**2. 实现 write 和 read**

```
write(name, content):
  查找当前目录 entries，找到 name 对应的 inode 号
  如果不存在或对应 inode 是目录：报错返回 -1
  inodes[num].content = content
  inodes[num].size    = content.size()
  打印：[FS] write: /路径/name, 写入 size 字节
  返回 size

read(name):
  查找当前目录 entries，找到 name 对应的 inode 号
  如果不存在：报错，返回空字符串
  打印：[FS] read: /路径/name, 读取 size 字节
  返回 inodes[num].content
```

**3. 修改 `main.cpp`，让进程运行时产生文件操作**

在调度演示之后，加一段模拟"进程执行并写日志"的场景：

```cpp
// 模拟进程执行：进程执行完后写一个日志文件
fs.cd("home");
fs.mkdir("logs");
fs.cd("logs");

// P1 完成后写日志
fs.touch("p1.log");
fs.write("p1.log", "[P1 init] 进程执行完毕，退出码 0");

// P2 完成后写日志
fs.touch("p2.log");
fs.write("p2.log", "[P2 shell] 进程执行完毕，退出码 0");

// 读回日志
std::string log = fs.read("p1.log");
std::cout << "读取日志：" << log << "\n";

fs.cdUp(); fs.cdUp();
fs.printTree();
```

### 检查点
- [ ] `write` 正确更新 inode 的 content 和 size
- [ ] `read` 能读回刚才写入的内容
- [ ] `stat` 能显示文件的 inode 号、大小、权限
- [ ] 文件系统操作日志在终端清晰可读

## 实验四：磁盘调度算法

### 目标
实现磁盘调度器，收集一批磁盘读请求（模拟进程读文件时产生），用 FCFS / SSTF / SCAN 三种算法排序服务，打印磁头移动轨迹和总移动距离。

### 背景知识：为什么要调度磁盘请求？

机械硬盘的磁头寻道（移动到目标磁道）是磁盘访问中最耗时的部分。如果不加优化，按请求到来顺序服务，磁头可能来回大幅移动。

调度的目标：**安排请求的服务顺序，让磁头总移动距离最小**。

三种算法：

| 算法 | 策略 | 特点 |
|------|------|------|
| FCFS | 按请求到来顺序服务 | 公平，但移动距离大 |
| SSTF（最短寻道时间优先）| 每次选距离磁头最近的请求 | 移动距离小，但可能饥饿 |
| SCAN（电梯算法）| 磁头向一方向移动，服务沿途请求，到端点再折返 | 均匀，无饥饿 |

### 步骤

**1. 新建 `src/io/disk_scheduler.h`**

```
class DiskScheduler {
public:
    // 初始参数：磁道总数、磁头初始位置
    DiskScheduler(int num_tracks, int initial_head);

    // 添加一个磁盘请求（磁道号）
    void addRequest(int track);

    // 运行指定算法，返回总移动距离
    int run(const std::string& algorithm, int direction = 1);
    // direction: 1=向高磁道, -1=向低磁道（SCAN 初始方向）

    // 打印磁头移动轨迹
    void printTrajectory() const;

private:
    int num_tracks;
    int initial_head;
    std::vector<int> requests;      // 原始请求列表（按到来顺序）
    std::vector<int> service_order; // 运行后的服务顺序
};
```

**2. 新建 `src/io/disk_scheduler.cpp`**

实现 `run()`，根据 algorithm 分支：

**FCFS：**
```
service_order = requests（直接按到来顺序）
计算总移动距离：相邻两个磁道号之差的绝对值之和
```

**SSTF：**
```
current = initial_head
while 还有未服务的请求:
    找距离 current 最近的未服务请求 next
    service_order.push_back(next)
    移动距离 += |next - current|
    current = next
```

**SCAN：**
```
将请求按磁道号排序

分成两组：
  higher：大于等于 initial_head 的（向高磁道方向的）
  lower： 小于 initial_head 的（向低磁道方向的）

如果 direction = 1（先向高磁道）：
  service_order = higher（升序）+ [到达最高端点] + lower（降序）

计算总移动距离
```

实现 `printTrajectory`：
```
打印：初始磁头位置 → 每个服务的磁道号 → 总移动距离

格式：
磁头轨迹：53 → 65 → 67 → 98 → 122 → 124 → 183 → (199端点) → 37 → 14
总移动距离：208 磁道
```

**3. 在 `main.cpp` 中演示三种算法对比**

```cpp
DiskScheduler disk(200, 53);   // 200个磁道，磁头在53
// 添加一组请求（经典教材例子）
for (int t : {98, 183, 37, 122, 14, 124, 65, 67}) {
    disk.addRequest(t);
}

std::cout << "\n--- FCFS ---\n";
disk.run("FCFS");
disk.printTrajectory();

std::cout << "\n--- SSTF ---\n";
disk.run("SSTF");
disk.printTrajectory();

std::cout << "\n--- SCAN ---\n";
disk.run("SCAN", 1);
disk.printTrajectory();
```

### 预期输出（教材标准答案）

```
--- FCFS ---
轨迹：53→98→183→37→122→14→124→65→67
总移动：640 磁道

--- SSTF ---
轨迹：53→65→67→37→14→98→122→124→183
总移动：236 磁道

--- SCAN ---
轨迹：53→65→67→98→122→124→183→(199)→37→14
总移动：208 磁道
```

### 检查点
- [ ] FCFS 总移动距离为 640
- [ ] SSTF 每步选择距离当前磁头最近的请求
- [ ] SCAN 正确分组后先服务一侧，再折返服务另一侧
- [ ] 三种算法结果与手算（或教材）一致

## 常见问题

**Q：`std::map<string, int>` 按什么顺序遍历？**  
A：按 key（文件名）的字典序（字母序）。如果你想按插入顺序，用 `std::unordered_map`。目录内容通常按字母序显示，所以 `map` 更合适。

**Q：`printTree` 怎么做树形缩进？**  
A：递归函数加一个 `prefix` 参数，每进入一层追加 `"│   "` 或 `"    "`，最后一项用 `"└── "`，非最后项用 `"├── "`。参考 Linux 的 `tree` 命令输出格式即可。

**Q：SCAN 到达端点时，端点需要是实际的最大磁道号吗？**  
A：是的。SCAN 算法中磁头移动到最后一个请求之后，还要继续移动到端点（如 track 199）才折返。如果不到端点就折返（只到最远请求）叫做 LOOK 算法，效率更高但逻辑略复杂，可选实现。
