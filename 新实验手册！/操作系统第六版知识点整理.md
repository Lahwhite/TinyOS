## 第一章：操作系统概述

### 1.1 操作系统的定义与目标

- **定义**：操作系统是控制和管理计算机硬件与软件资源，并为用户和应用程序提供便利接口的系统软件。
- **目标**：
  - **方便性**：使计算机更便于用户使用
  - **有效性**：使硬件以更高效的方式使用
  - **扩展性**：便于对系统进行构造、测试和功能扩充

### 1.2 操作系统的地位

```
用户程序（应用软件）
     ↓
 操作系统（系统软件）
     ↓
  硬件（CPU、内存、I/O 设备）
```

操作系统是用户与硬件之间的**接口**，也是计算机系统中最基本的**系统软件**。

### 1.3 操作系统的功能

| 功能 | 描述 |
|------|------|
| **进程管理** | 进程的创建、调度、同步、通信、终止 |
| **内存管理** | 内存分配与回收、地址映射、内存保护与扩充 |
| **文件管理** | 文件的存储、检索、共享与保护 |
| **设备管理** | I/O 设备的分配、驱动、缓冲与控制 |
| **用户接口** | 命令行、图形界面、系统调用接口 |

### 1.4 操作系统的类型

| 类型 | 特征 | 典型系统 |
|------|------|----------|
| **批处理操作系统** | 作业成批处理，无需用户干预 | 早期 IBM 大型机 |
| **分时操作系统** | 多用户共享 CPU，轮流使用时间片 | UNIX |
| **实时操作系统** | 在规定时间内完成响应 | VxWorks、FreeRTOS |
| **分布式操作系统** | 多台计算机协同工作 | Amoeba |
| **网络操作系统** | 管理网络资源，提供网络服务 | Windows Server |
| **个人计算机操作系统** | 单用户，提供友好界面 | Windows、macOS |
| **嵌入式操作系统** | 针对特定硬件，资源受限 | Android、iOS |

### 1.5 操作系统的发展历程

1. **无操作系统（1945-1955）**：人工操作，机器语言编程
2. **批处理系统（1955-1965）**：作业成批提交，FMS、IBSYS
3. **多道程序系统（1965-1980）**：多道作业并发执行，IBM OS/360
4. **分时系统（1970s）**：CTSS、MULTICS、UNIX
5. **个人计算机时代（1980s）**：MS-DOS、Windows、Macintosh
6. **网络与分布式（1990s-至今）**：Linux、Windows NT、分布式系统

### 1.6 计算机系统结构基础

#### 1.6.1 计算机系统的组成

- **CPU**：运算器 + 控制器
- **内存（主存）**：存放运行中的程序和数据
- **I/O 设备**：键盘、鼠标、磁盘、显示器
- **总线**：连接各部件的数据通道（数据总线、地址总线、控制总线）

#### 1.6.2 中断机制

- **中断**：I/O 完成或异常事件时，CPU 暂停当前任务，转去处理中断
- **中断类型**：
  - **外部中断（硬中断）**：来自 I/O 设备或时钟
  - **内部中断（异常/陷阱）**：除零错误、页缺失、系统调用
- **中断处理流程**：
  1. 保存现场（PC、寄存器）
  2. 转向中断处理程序
  3. 处理中断
  4. 恢复现场
  5. 返回断点继续执行

#### 1.6.3 系统调用

- **定义**：用户程序通过系统调用请求操作系统服务（从用户态进入内核态）
- **常见系统调用类别**：进程控制、文件操作、设备管理、信息维护、通信
- **工作流程**：
  ```
  用户程序 → 执行 int/syscall 指令 → 进入内核态 → 执行系统调用例程 → 返回用户态
  ```

#### 1.6.4 用户态与内核态

| | 用户态（User Mode）| 内核态（Kernel Mode）|
|---|---|---|
| 权限 | 受限，只能执行非特权指令 | 全权，可执行所有指令 |
| 地址空间 | 用户空间 | 全部地址空间 |
| 切换时机 | 系统调用、中断、异常 | 返回时 |

### 1.7 操作系统的结构

#### 1.7.1 单体结构（宏内核）

- 所有功能集中在一个大程序中运行在内核态
- 优点：高效；缺点：难以维护，一个模块出错影响整个系统

#### 1.7.2 层次结构

- 将 OS 组织为分层结构，每层使用下层提供的服务
- 代表：THE 系统（Dijkstra）

#### 1.7.3 微内核结构

- 只有核心功能（进程调度、内存管理、进程间通信）运行在内核态
- 其余功能以用户进程方式运行
- 优点：可靠性高、易于扩展；缺点：性能开销大（频繁切换）
- 代表：MINIX、Mach

#### 1.7.4 模块化结构

- 内核由可动态加载的模块组成
- 代表：Linux（loadable kernel modules）

#### 1.7.5 虚拟机结构

- 使用 Hypervisor（虚拟机监视器）在物理硬件上创建多个虚拟机
- 类型1（裸机型）：VMware ESXi；类型2（宿主型）：VirtualBox

## 第二章：进程管理

### 2.1 进程的概念

- **程序**：静态的，存储在磁盘上的可执行文件
- **进程（Process）**：程序的一次**执行过程**，是系统进行**资源分配**和**调度**的基本单位
- **进程的组成**：
  - **程序代码**（文本段）
  - **程序计数器（PC）**：当前执行位置
  - **寄存器**：CPU 寄存器内容
  - **栈（Stack）**：临时数据（局部变量、函数参数）
  - **堆（Heap）**：动态分配的内存
  - **数据段**：全局变量

### 2.2 进程状态与转换

#### 五状态模型

```
               admit
  New ──────────────────→ Ready
                           ↑  ↓ dispatch
                 I/O完成   │  ↓
  Blocked ←─────────────── Running ──────────→ Exit/Terminated
      │              I/O请求  
      └─────────────────────────────────────→ Ready（时间片到）
```

| 状态 | 说明 |
|------|------|
| **新建（New）** | 进程正在被创建 |
| **就绪（Ready）** | 进程已准备好，等待 CPU |
| **运行（Running）** | 进程占用 CPU 执行 |
| **阻塞/等待（Blocked）** | 进程等待某事件（I/O 完成、信号等） |
| **终止（Terminated）** | 进程执行完毕 |

#### 七状态模型（引入挂起）

新增两个状态：
- **就绪挂起（Ready-Suspended）**：就绪状态的进程被换出到外存
- **阻塞挂起（Blocked-Suspended）**：阻塞状态的进程被换出到外存

### 2.3 进程控制块（PCB）

**进程控制块（Process Control Block, PCB）** 是进程存在的唯一标志，包含进程的所有管理信息。

| 信息类别 | 具体内容 |
|----------|----------|
| **进程标识** | PID、父进程 PID、用户 ID |
| **CPU 状态信息** | 寄存器值、PC、程序状态字（PSW） |
| **进程调度信息** | 状态、优先级、调度参数 |
| **内存管理信息** | 页表/段表基址、内存限制 |
| **I/O 状态信息** | 已分配 I/O 设备、打开文件列表 |
| **记账信息** | CPU 使用时间、开始时间 |

### 2.4 进程控制

#### 进程创建

- **步骤**：
  1. 申请空白 PCB
  2. 分配必要资源（内存、文件等）
  3. 初始化 PCB
  4. 将进程插入就绪队列
- **UNIX/Linux**：`fork()` 创建子进程（复制父进程），`exec()` 加载新程序
- **Windows**：`CreateProcess()`

#### 进程终止

- **正常终止**：`exit()` 系统调用
- **异常终止**：除零错误、访问越界、被杀死（`kill()`）
- 子进程终止后变为**僵尸进程（Zombie）**，等待父进程通过 `wait()` 回收

#### 进程阻塞与唤醒

- **阻塞**：进程主动调用 `block()` 原语，将自身置于阻塞状态
- **唤醒**：其他进程调用 `wakeup()` 原语，将阻塞进程移入就绪队列

### 2.5 进程调度

#### 调度层次

| 层次 | 名称 | 功能 |
|------|------|------|
| **高级调度（作业调度）** | 长程调度 | 从外存调入内存，决定哪些作业进入系统 |
| **中级调度（内存调度）** | 中程调度 | 决定哪些进程挂起/激活（交换） |
| **低级调度（进程调度）** | 短程调度 | 从就绪队列选择进程分配 CPU |

#### 调度目标

- **公平性**：各进程合理分享 CPU
- **高吞吐量**：单位时间完成更多进程
- **低响应时间**：交互式系统响应快
- **CPU 利用率**：保持 CPU 忙碌
- **低周转时间**：从提交到完成时间短

#### 调度算法

##### 2.5.1 先来先服务（FCFS，First Come First Served）

- 按到达就绪队列的顺序分配 CPU
- **特点**：非抢占式，简单公平
- **缺点**：**护航效应**（大进程独占 CPU，小进程长时间等待）
- **适用**：批处理系统

##### 2.5.2 最短作业优先（SJF，Shortest Job First）

- 选择预计执行时间最短的进程
- 分为：非抢占式（SJF）和抢占式（**SRTF**，最短剩余时间优先）
- **优点**：平均等待时间最短（最优）
- **缺点**：难以预知 CPU 执行时间，可能导致**饥饿**

##### 2.5.3 优先级调度（Priority Scheduling）

- 选择优先级最高的进程运行
- **抢占式**：高优先级进程可抢占低优先级
- **缺点**：低优先级进程可能**饥饿**
- **解决**：**老化（Aging）**——随时间推移提升低优先级进程的优先级

##### 2.5.4 轮转调度（RR，Round Robin）

- 每个进程分配一个**时间片（Quantum）**，用完时间片则抢占
- **时间片大小的影响**：
  - 时间片过大 → 退化为 FCFS
  - 时间片过小 → 上下文切换开销过大
- **适用**：分时系统

##### 2.5.5 多级队列调度（Multi-level Queue）

- 将就绪队列分为多个子队列（前台/后台）
- 各队列有各自的调度算法
- 队列间采用优先级调度或时间分配

##### 2.5.6 多级反馈队列调度（Multi-level Feedback Queue，MLFQ）

- 进程根据 CPU 使用情况动态移动队列
- 新进程进入最高优先级队列（短时间片）
- 用完时间片未完成 → 降入低优先级队列（长时间片）
- **优点**：短作业优先，同时不饿死长作业；自适应性强

#### 调度评估指标

| 指标 | 公式 |
|------|------|
| **CPU 利用率** | CPU 忙碌时间 / 总时间 |
| **吞吐量** | 完成进程数 / 时间 |
| **周转时间** | 完成时间 - 提交时间 |
| **等待时间** | 周转时间 - 执行时间 |
| **响应时间** | 第一次响应时间 - 提交时间 |

### 2.6 进程间通信（IPC）

#### 2.6.1 共享内存（Shared Memory）

- 多个进程共享同一块内存区域，直接读写通信
- **优点**：速度快（无需内核介入）
- **缺点**：需要同步机制防止竞争条件

#### 2.6.2 消息传递（Message Passing）

- 通过 `send(message)` 和 `receive(message)` 原语通信
- 分类：
  - **直接通信**：进程间直接命名
  - **间接通信**：通过邮箱/消息队列
  - **同步/阻塞**：发送/接收时阻塞等待
  - **异步/非阻塞**：发送后继续执行

#### 2.6.3 管道（Pipe）

- 单向通信的字节流信道
- **匿名管道**：`pipe()`，只能用于父子进程间
- **命名管道（FIFO）**：`mkfifo()`，无关系进程也可使用

#### 2.6.4 套接字（Socket）

- 网络通信的基本接口，可跨机器通信
- 也支持本地 IPC（Unix Domain Socket）

#### 2.6.5 信号（Signal）

- 异步通知机制，通知进程某事件发生
- 例：`SIGKILL`（终止）、`SIGINT`（Ctrl+C）、`SIGALRM`（定时器）

## 第三章：线程与并发

### 3.1 线程的概念

- **线程（Thread）**：CPU 调度的基本单位，进程内的执行流
- **一个进程可以包含多个线程**，共享进程的资源（代码段、数据段、文件描述符）
- **每个线程独有**：线程 ID、PC、寄存器、栈

### 3.2 进程与线程的比较

| 比较项 | 进程 | 线程 |
|--------|------|------|
| 资源分配 | 是基本单位 | 共享进程资源 |
| 调度单位 | 是（传统） | 是（现代） |
| 切换开销 | 大（需切换地址空间） | 小 |
| 通信方式 | IPC（复杂） | 共享内存（简单但需同步） |
| 独立性 | 高 | 低（一个线程崩溃影响整个进程） |
| 创建开销 | 大 | 小 |

### 3.3 多线程模型

| 模型 | 说明 |
|------|------|
| **多对一（Many-to-One）** | 多个用户线程映射到一个内核线程；并发度低，一个线程阻塞导致全部阻塞 |
| **一对一（One-to-One）** | 每个用户线程对应一个内核线程；并发度高，但线程创建代价大（Linux、Windows 使用此模型） |
| **多对多（Many-to-Many）** | 多个用户线程映射到多个内核线程；灵活高效（Solaris 早期版本） |

### 3.4 线程库

- **POSIX Pthreads**：UNIX/Linux 平台标准线程 API
- **Windows Threads**：Win32 API
- **Java Threads**：通过 JVM 实现（底层依赖操作系统）

### 3.5 并发与同步

#### 3.5.1 竞争条件（Race Condition）

多个进程/线程并发访问共享数据，且最终结果依赖于执行顺序，导致数据不一致。

#### 3.5.2 临界区（Critical Section）

- **临界资源**：每次只允许一个进程访问的资源
- **临界区**：访问临界资源的那段代码
- **临界区问题的解决条件**：
  1. **互斥（Mutual Exclusion）**：同时只有一个进程在临界区
  2. **前进（Progress）**：不在临界区的进程不得阻止他人进入
  3. **有限等待（Bounded Waiting）**：等待进入临界区的时间有限

#### 3.5.3 Peterson 算法

经典的软件解决方案（针对两个进程的临界区问题）：

```c
// 进程 P0
flag[0] = true;
turn = 1;
while (flag[1] && turn == 1);
// 临界区
flag[0] = false;
```

满足互斥、前进、有限等待三个条件，但在现代处理器上可能由于指令重排而失效。

#### 3.5.4 硬件同步原语

- **禁用中断**：简单，但不适用于多核
- **Test-and-Set（TAS）**：原子操作，读取并设置锁
- **Compare-and-Swap（CAS）**：比较值并原子交换，用于实现无锁数据结构

### 3.6 互斥量（Mutex）

- **互斥锁（Mutex Lock）**：保护临界区的锁，`acquire()` 获取，`release()` 释放
- **自旋锁（Spinlock）**：等待时忙等（循环检查），适合等待时间极短的场景

### 3.7 信号量（Semaphore）

- **整型信号量**：`S` 为整数，`P/wait()` 减1（若 S<0 则阻塞），`V/signal()` 加1（若有进程等待则唤醒）
- **记录型信号量**：包含整数值和等待队列

```
wait(S):            signal(S):
  S.value--;          S.value++;
  if S.value < 0      if S.value <= 0
    block();            wakeup(next);
```

- **二值信号量（Binary Semaphore）**：值只有0或1，等价于互斥锁
- **计数信号量（Counting Semaphore）**：可用于管理多个相同资源

#### 信号量应用

| 应用 | 实现方式 |
|------|----------|
| **互斥**（临界区保护） | 初始化为1的信号量 |
| **同步**（执行顺序约束） | 初始化为0的信号量 |
| **资源管理** | 初始化为资源数量的信号量 |

### 3.8 经典同步问题

#### 3.8.1 生产者-消费者问题（有界缓冲区）

```
信号量：
  mutex = 1（互斥访问缓冲区）
  empty = n（空槽数量，初始=n）
  full  = 0（满槽数量，初始=0）

生产者：
  wait(empty); wait(mutex);
  // 放入产品
  signal(mutex); signal(full);

消费者：
  wait(full); wait(mutex);
  // 取出产品
  signal(mutex); signal(empty);
```

#### 3.8.2 读者-写者问题

- **规则**：多个读者可并发读；写者写时独占
- **读者优先**：只要有读者，写者等待（可能使写者饥饿）
- **写者优先**：有写者等待时，新读者也要等待

#### 3.8.3 哲学家进餐问题

- **问题**：5个哲学家围坐，5根筷子（每人需左右各一根），可能发生死锁
- **解决方案**：
  1. 最多允许4个哲学家同时拿筷子
  2. 奇数哲学家先取左，偶数哲学家先取右
  3. 使用管程，只有两根筷子都可用时才允许取

### 3.9 管程（Monitor）

- **定义**：包含共享数据、操作过程和初始化代码的高级同步原语
- **特点**：同一时刻只有一个进程在管程内活动（自带互斥）
- **条件变量（Condition Variable）**：`wait()` 和 `signal()` 用于进程在管程内等待/唤醒

## 第四章：内存管理

### 4.1 内存管理的目标

- **地址空间保护**：防止进程间相互干扰
- **地址映射**：逻辑地址 → 物理地址
- **内存分配与回收**：高效管理内存空间
- **内存共享**：允许进程共享公共代码/数据
- **内存扩充**：通过虚拟内存技术扩大可用空间

### 4.2 地址绑定

程序中的地址（符号地址）需要映射到内存物理地址，可发生在三个时机：

| 时机 | 说明 |
|------|------|
| **编译时** | 编译时已知内存位置，生成绝对地址（.com 文件） |
| **加载时** | 加载到内存时确定地址（可重定位代码） |
| **执行时** | 运行期间动态绑定（需硬件支持，现代 OS 普遍使用） |

### 4.3 逻辑地址与物理地址

- **逻辑地址（虚拟地址）**：CPU 产生的地址，程序员使用
- **物理地址**：内存硬件使用的实际地址
- **内存管理单元（MMU）**：将逻辑地址转换为物理地址的硬件

### 4.4 连续内存分配

#### 4.4.1 单一连续分配

- 内存分为系统区和用户区，整个用户区给一个进程
- 只适合单道程序系统

#### 4.4.2 固定分区分配

- 将内存划分为若干固定大小的分区
- **内部碎片**：分区内未被使用的空间
- 每分区只能装一个进程

#### 4.4.3 动态分区分配

- 按照进程实际需求动态创建分区
- **外部碎片**：分区间的零散空闲空间
- **分配算法**：

| 算法 | 策略 | 优缺点 |
|------|------|--------|
| **首次适应（First Fit）** | 从头找第一个足够大的空闲区 | 快速，但低地址碎片多 |
| **最佳适应（Best Fit）** | 找最小的足够大的空闲区 | 外部碎片最小，但搜索慢 |
| **最坏适应（Worst Fit）** | 找最大的空闲区 | 留下大块，但大进程难分配 |
| **循环首次适应（Next Fit）** | 从上次位置继续搜索 | 分布均匀 |

- **碎片整理（压缩）**：移动已分配区域，合并外部碎片，代价高

### 4.5 分页（Paging）

#### 基本概念

- **页（Page）**：逻辑地址空间划分为固定大小的块（2的幂次方，如 4KB）
- **帧/页框（Frame）**：物理内存划分为同样大小的块
- **页表（Page Table）**：记录页与帧的映射关系

#### 地址转换

```
逻辑地址 = 页号(p) + 页内偏移(d)
         ↓ 查页表
物理地址 = 帧号(f) + 页内偏移(d)
```

#### 硬件支持

- **页表基址寄存器（PTBR）**：存储页表起始地址
- **快表（TLB，Translation Lookaside Buffer）**：
  - 小型硬件缓存，存储常用的页表项
  - **命中（Hit）**：直接得到帧号，无需访问内存
  - **未命中（Miss）**：访问内存中的页表，并将结果存入 TLB

#### 有效内存访问时间（EAT）

```
EAT = α × (TLB查找时间 + 内存访问时间)
    + (1-α) × (TLB查找时间 + 2×内存访问时间)
```
（α 为 TLB 命中率）

#### 分页的保护与共享

- **保护位**：页表中每个条目包含读/写/执行保护位
- **有效/无效位**：标记页是否在进程地址空间内
- **共享页**：多个进程的页表项可指向同一帧（代码共享）

#### 多级页表

- 将页表本身也分页（两级、三级、四级）
- **节省空间**：稀疏地址空间不需要分配所有页表内存
- 例：x86-64 使用四级页表（PML4 → PDPT → PD → PT）

#### 反向页表（Inverted Page Table）

- 为每个物理帧记录一个条目（帧号 → 进程ID + 逻辑页号）
- 节省内存，但搜索慢（通常配合哈希）

### 4.6 分段（Segmentation）

- 将程序按**逻辑单元**划分（代码段、数据段、栈段等）
- **段表**：段号 → (段基址, 段长度)
- **优点**：符合程序逻辑结构，便于保护和共享
- **缺点**：产生外部碎片，需要压缩

#### 段页式（Segmentation with Paging）

- 先分段，再对每段进行分页
- 综合两者优点：逻辑清晰 + 无外部碎片
- 段表中每个条目指向该段的页表基址
- Intel x86 即采用段页式结构

## 第五章：虚拟内存

### 5.1 虚拟内存的概念

- **虚拟内存**：允许进程的逻辑地址空间大于物理内存
- **基本思想**：只将程序**当前需要**的部分加载到内存（**按需调页**）
- **优点**：
  - 提高内存利用率
  - 允许更多进程并发运行
  - 程序可以大于物理内存

### 5.2 按需调页（Demand Paging）

- 页面初始时不加载，访问时若不在内存则触发**缺页中断（Page Fault）**
- **缺页中断处理流程**：
  1. 内部表检查：是否是合法内存访问
  2. 若合法，找到一个空闲帧
  3. 从磁盘将缺少的页读入该帧
  4. 更新页表（设置有效位）
  5. 重新执行引发缺页的指令

### 5.3 页面置换算法

当内存满时，需选择一个页面淘汰（换出到磁盘）：

#### 5.3.1 最优置换（OPT，Optimal）

- 淘汰**将来最长时间不被使用**的页
- **理论最优**（缺页率最低），但无法实现（不知道未来）
- 用作评估其他算法的基准

#### 5.3.2 先进先出（FIFO）

- 淘汰最先进入内存的页
- 简单，但性能差
- **Bélády 异常**：分配更多帧时，FIFO 的缺页率反而可能升高

#### 5.3.3 最近最少使用（LRU，Least Recently Used）

- 淘汰最长时间未被访问的页（基于时间局部性）
- 性能接近 OPT，无 Bélády 异常
- **实现方式**：
  - **计数器**：每次访问时记录时间戳
  - **栈**：访问时将页移到栈顶，淘汰栈底

#### 5.3.4 时钟算法（Clock / Second Chance）

- 近似 LRU 的高效实现
- 每页有一个**引用位（R）**，被访问时置1
- 选择淘汰页时顺时针扫描：R=0 则淘汰；R=1 则清零并继续

#### 5.3.5 改进时钟算法（NRU，Not Recently Used）

- 综合**引用位（R）**和**修改位（M，脏位）**
- 优先淘汰 (R=0, M=0) 的页，其次 (R=0, M=1)，再次 (R=1, M=0)

### 5.4 帧分配策略

- **固定分配**：每个进程分配固定数量的帧
- **可变分配**：根据进程行为动态调整帧数
- **局部置换**：只从当前进程的帧中置换
- **全局置换**：可从任何进程的帧中置换（更高吞吐量，但单进程行为不可控）

### 5.5 抖动（Thrashing）

- **现象**：进程大部分时间用于换页，几乎没有实际执行时间
- **原因**：分配给进程的帧数太少，频繁发生缺页
- **解决方案**：
  - **工作集模型（Working Set Model）**：监控进程最近 Δ 时间内访问的页集合，保证工作集全部在内存
  - **缺页率控制（PFF）**：根据缺页率动态增减帧数

### 5.6 内存映射文件

- 将文件映射到进程地址空间，像访问内存一样访问文件
- `mmap()` 系统调用（Linux/macOS）
- 减少 I/O 系统调用开销，文件共享高效

### 5.7 内核内存分配

- **伙伴系统（Buddy System）**：将内存按2的幂次方大小分配，便于合并
- **Slab 分配器**：为内核对象（task_struct 等）预分配内存缓存，减少碎片

## 第六章：文件系统

### 6.1 文件的概念

- **文件（File）**：存储在外存上的有名字的相关信息集合
- **文件的属性**：名字、类型、大小、位置、创建/修改时间、保护（权限）
- **文件类型**：
  - 按内容：普通文件、目录文件、特殊文件（设备文件）
  - 按访问方式：顺序访问、随机访问（直接访问）

### 6.2 文件系统接口

#### 系统调用

| 操作 | POSIX 调用 |
|------|------------|
| 创建 | `creat()`, `open(O_CREAT)` |
| 打开 | `open()` |
| 读写 | `read()`, `write()` |
| 定位 | `lseek()` |
| 关闭 | `close()` |
| 删除 | `unlink()` |
| 信息 | `stat()` |

#### 文件描述符（File Descriptor）

- 内核为每个打开的文件分配整数句柄（fd）
- 进程维护文件描述符表，指向系统文件表，再指向 inode

### 6.3 目录结构

| 结构 | 说明 |
|------|------|
| **单级目录** | 所有文件在同一目录，简单但易冲突 |
| **两级目录** | 每个用户有独立目录，解决命名冲突 |
| **树形目录** | 层次结构，支持路径（绝对/相对），UNIX/Windows 使用 |
| **无环图目录** | 允许共享文件/目录（硬链接/符号链接），需引用计数 |

### 6.4 文件系统实现

#### 6.4.1 磁盘布局

典型磁盘分区布局：
```
| 引导块 | 超级块 | 空闲块位图 | inode 表 | 数据块 |
```

- **超级块**：文件系统元信息（块大小、inode 数量、空闲块信息等）
- **inode（索引节点）**：存储文件元数据，不含文件名

#### 6.4.2 文件存储结构

| 方式 | 说明 | 优缺点 |
|------|------|--------|
| **连续分配** | 文件占连续磁盘块 | 顺序访问快，外部碎片，难扩展 |
| **链接分配** | 磁盘块通过指针链接 | 无外部碎片，随机访问慢，可靠性差 |
| **FAT（文件分配表）** | 链接分配变体，将指针存在表中 | 随机访问稍好，FAT 本身需缓存 |
| **索引分配（inode）** | 索引块集中存储所有块指针 | 随机访问好，inode 有大小限制 |

#### Unix inode 结构（多级索引）

```
inode:
  ├── 12个直接块指针        → 直接指向数据块
  ├── 1个一级间接块指针     → 指向包含256个指针的块
  ├── 1个二级间接块指针     → 两层指针
  └── 1个三级间接块指针     → 三层指针
```

### 6.5 空闲空间管理

| 方法 | 说明 |
|------|------|
| **位图（Bitmap）** | 每个块用一位标记（0=空闲，1=已用） |
| **空闲链表** | 空闲块用指针链接 |
| **成组链接** | 将空闲块分组，提高效率（UNIX 使用） |

### 6.6 文件系统性能优化

- **缓冲区高速缓存（Buffer Cache）**：将常用磁盘块缓存在内存
- **预读取（Read-Ahead）**：预测并提前加载可能访问的块
- **延迟写入（Write-Back）**：批量异步写入（提高性能但断电有风险）
- **日志文件系统（Journaling）**：记录操作日志，断电后可恢复（ext3/4、NTFS）

### 6.7 文件保护

#### 访问控制列表（ACL）

- 每个文件维护能访问它的用户/组列表及其权限

#### UNIX 权限位

```
-rwxr-xr-- 1 owner group size date filename
```

- **r**（读）、**w**（写）、**x**（执行）
- 分三组：**所有者**、**组**、**其他用户**

## 第七章：I/O 系统

### 7.1 I/O 硬件

#### 设备分类

| 类型 | 特征 | 例子 |
|------|------|------|
| **块设备** | 按块读写，可随机访问 | 磁盘、SSD |
| **字符设备** | 按字节流读写，顺序访问 | 键盘、鼠标、串口 |
| **网络设备** | 特殊类别，通过套接字访问 | 网卡 |

#### I/O 端口与内存映射 I/O

- **I/O 端口**：独立的 I/O 地址空间（`in/out` 指令）
- **内存映射 I/O（MMIO）**：设备寄存器映射到物理内存地址，用普通读写指令访问

### 7.2 I/O 控制方式

| 方式 | 说明 | 效率 |
|------|------|------|
| **程序控制 I/O（轮询）** | CPU 忙等，反复检查设备状态 | 最低，浪费 CPU |
| **中断驱动 I/O** | I/O 完成时向 CPU 发中断，CPU 继续做其他事 | 较好 |
| **直接内存访问（DMA）** | DMA 控制器直接在内存和设备间传输数据，CPU 只在开始和结束时介入 | 最高 |

### 7.3 I/O 软件层次

```
用户进程（I/O 调用）
    ↓
I/O 库（标准库 printf 等）
    ↓
设备无关 I/O 软件（统一接口、缓冲、错误处理）
    ↓
设备驱动程序（设备相关，与硬件交互）
    ↓
中断处理程序
    ↓
硬件
```

### 7.4 磁盘结构

- **磁盘**：由多个**盘片（Platter）**组成
- **磁道（Track）**：盘片上的圆形轨道
- **扇区（Sector）**：磁道上的最小读写单位（512B 或 4KB）
- **柱面（Cylinder）**：所有盘片上相同磁道的集合

#### 磁盘访问时间

```
总时间 = 寻道时间 + 旋转延迟 + 传输时间
```
- **寻道时间**：磁头移动到目标磁道（最贵，毫秒级）
- **旋转延迟**：等待目标扇区转到磁头下方（平均半圈时间）
- **传输时间**：实际数据传输

### 7.5 磁盘调度算法

| 算法 | 策略 | 特点 |
|------|------|------|
| **FCFS** | 按请求顺序服务 | 公平，但效率低 |
| **SSTF（最短寻道时间优先）** | 选最近磁道 | 效率高，但可能饥饿 |
| **SCAN（电梯算法）** | 向一方向移动，到头再折返 | 类似电梯，无饥饿 |
| **C-SCAN（循环扫描）** | 只向一个方向，到头后回到开始 | 均匀服务，更公平 |
| **LOOK/C-LOOK** | SCAN/C-SCAN 的改进，不到端点就折返 | 效率更高 |

### 7.6 I/O 缓冲

- **单缓冲**：I/O 和 CPU 交替使用单个缓冲区
- **双缓冲**：一个缓冲区给 CPU 处理，另一个给 I/O，提高并行性
- **循环缓冲（环形缓冲）**：多个缓冲区循环使用，生产者-消费者模型

### 7.7 RAID 技术

| 级别 | 说明 | 特点 |
|------|------|------|
| **RAID 0**（条带化） | 数据分散在多个磁盘，无冗余 | 速度最快，无容错 |
| **RAID 1**（镜像） | 完全复制到另一磁盘 | 最高可靠性，空间利用率50% |
| **RAID 5** | 分布式奇偶校验 | 平衡性能、容量、可靠性 |
| **RAID 10**（1+0） | 先镜像再条带 | 高性能+高可靠 |

## 第八章：死锁

### 8.1 死锁的概念

**死锁（Deadlock）**：一组进程中，每个进程都在等待该组中其他进程所占有的资源，导致所有进程永久阻塞。

### 8.2 死锁产生的四个必要条件

| 条件 | 说明 |
|------|------|
| **互斥（Mutual Exclusion）** | 资源一次只能被一个进程占用 |
| **占有并等待（Hold and Wait）** | 进程持有资源的同时等待其他资源 |
| **不可抢占（No Preemption）** | 资源只能被持有者自愿释放 |
| **循环等待（Circular Wait）** | 进程间存在资源等待的环形链 |

**四个条件同时成立才会发生死锁！**

### 8.3 资源分配图（RAG）

- **进程节点**（圆形）和**资源节点**（方形）
- **请求边**：进程 → 资源（进程请求该资源）
- **分配边**：资源 → 进程（资源已分配给进程）

**判断死锁**：
- 若图中**无环**：无死锁
- 若图中**有环**：
  - 每种资源只有一个实例 → 必定死锁
  - 每种资源有多个实例 → 可能死锁

### 8.4 死锁处理策略

| 策略 | 方法 |
|------|------|
| **死锁预防（Prevention）** | 破坏四个必要条件之一 |
| **死锁避免（Avoidance）** | 动态分配时确保系统不进入不安全状态 |
| **死锁检测（Detection）** | 允许死锁发生，定期检测并解除 |
| **鸵鸟策略（Ignore）** | 忽略死锁（大多数 OS 的实际做法）|

### 8.5 死锁预防

#### 破坏互斥条件

- 对可共享资源（如只读文件）不需要互斥 → 无法破坏（有些资源必须互斥）

#### 破坏占有并等待条件

- **方法1**：进程开始前请求所有资源（资源利用率低）
- **方法2**：请求新资源时必须先释放当前所有资源

#### 破坏不可抢占条件

- 若进程请求资源失败，则释放其当前所有资源（资源被"抢占"）
- 适用于可保存状态的资源（CPU 寄存器、内存），不适合打印机等

#### 破坏循环等待条件

- **资源有序分配**：对所有资源类型排序，进程必须按升序请求资源
- 这样不可能形成循环

### 8.6 死锁避免

#### 安全状态与安全序列

- **安全状态**：存在至少一个**安全序列**，使所有进程都能完成
- **安全序列 \(<P_1, P_2, ..., P_n>\)**：对每个 \(P_i\)，其资源需求可由当前可用资源 + \(P_1...P_{i-1}\) 释放的资源满足
- 安全状态 → 不会死锁；不安全状态 → **可能**死锁

#### 银行家算法（Banker's Algorithm）

适用于**多个资源类、多个实例**的情况。

**数据结构**：
- `Available[m]`：各资源可用数量
- `Max[n][m]`：每个进程的最大资源需求
- `Allocation[n][m]`：当前已分配给每个进程的资源
- `Need[n][m]` = `Max - Allocation`：每个进程还需的资源

**安全算法**：

```
Work = Available;  Finish[i] = false for all i

loop:
  找 i 使得 Finish[i]=false 且 Need[i] ≤ Work
  若找到：Work += Allocation[i]; Finish[i] = true
  若所有 Finish[i]=true：安全状态
  否则：不安全
```

**资源请求算法**：进程 \(P_i\) 请求 \(Request_i\) 时：
1. `Request_i ≤ Need_i`（不超过声明需求）
2. `Request_i ≤ Available`（有足够资源）
3. 试探性分配，运行安全算法
4. 若安全则分配，否则等待

### 8.7 死锁检测与恢复

#### 检测算法（类似银行家）

```
Work = Available;  Finish[i] = (Allocation[i]==0)

找 i 使得 Finish[i]=false 且 Request[i] ≤ Work
  Work += Allocation[i]; Finish[i] = true

若存在 Finish[i]=false：P_i 处于死锁中
```

#### 死锁恢复

| 方法 | 说明 |
|------|------|
| **终止进程** | 终止所有死锁进程（代价大），或逐个终止直到解除死锁 |
| **资源抢占** | 抢占进程资源（选择代价最小的进程），需将进程回滚到安全点 |

## 知识点总结

### 核心概念速查

| 概念 | 关键点 |
|------|--------|
| **进程 vs 线程** | 进程：资源单位；线程：调度单位；线程共享进程资源 |
| **调度算法** | FCFS（公平）、SJF（最优平均等待）、RR（分时）、MLFQ（自适应） |
| **同步机制** | 信号量（P/V）、互斥锁、管程、条件变量 |
| **内存分页** | 消除外部碎片，TLB 加速地址转换 |
| **虚拟内存** | 按需调页，LRU 置换算法，注意抖动问题 |
| **文件系统** | inode 多级索引，目录层次结构，日志保证一致性 |
| **磁盘调度** | SCAN/C-SCAN 减少寻道时间 |
| **死锁** | 四个必要条件，银行家算法避免，检测+恢复 |

### 重要公式

| 公式 | 含义 |
|------|------|
| `周转时间 = 完成时间 - 到达时间` | 进程完整生命周期 |
| `带权周转时间 = 周转时间 / 服务时间` | 衡量调度公平性 |
| `EAT = α(1 + t_mem) + (1-α)(1 + 2t_mem)` | 含TLB的有效访问时间 |
| `缺页率 p = 缺页次数 / 总访问次数` | 虚拟内存性能指标 |
