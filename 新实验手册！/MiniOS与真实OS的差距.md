# MiniOS 与真实 OS 的差距

## 已实现的核心概念

| MiniOS 模块 | 对应真实 OS 概念 | 完整度 |
|------------|----------------|--------|
| PCB + 状态机 | 进程管理 | ⭐⭐⭐ 扎实 |
| FCFS/SJF/RR/Priority | CPU 调度算法 | ⭐⭐⭐ 扎实 |
| firstFit/bestFit | 动态内存分区 | ⭐⭐⭐ 扎实 |
| Semaphore + 生产者消费者 | 同步原语 | ⭐⭐ 基础完成 |
| inode + 目录树 | 文件系统结构 | ⭐⭐ 基础完成 |
| FCFS/SSTF/SCAN | 磁盘调度 | ⭐⭐⭐ 扎实 |
| 页表 + LRU | 虚拟内存 | ⭐⭐ 基础完成 |
| 银行家算法 | 死锁预防 | ⭐⭐ 基础完成 |

## 缺失的部分

### 【核心缺失，影响理解】

| 缺失内容 | 真实 OS 里的作用 | MiniOS 的简化方式 |
|---------|----------------|-----------------|
| **中断机制** | OS 的一切主动行为（调度、I/O完成通知）都靠中断驱动，时钟中断强制切换进程 | 调度是手动调用函数，没有"强制抢占" |
| **用户态/内核态** | 保护机制基础，进程不能随意访问硬件和他人内存（CPU Ring 0 / Ring 3） | 所有代码在同等权限下运行 |
| **上下文切换** | 切换进程时保存/恢复寄存器、栈指针等完整 CPU 状态 | "调度"只改了 PCB 里的 state 字段 |

### 【重要缺失，影响完整性】

| 缺失内容 | 真实 OS 里的作用 |
|---------|----------------|
| **Page Cache** | 文件读写先过内存缓存，大部分 I/O 不需要真正访问磁盘 |
| **块设备层** | 文件块号 → 物理扇区号的映射，连接文件系统和磁盘调度 |
| **文件描述符表** | 每个进程维护自己打开的文件列表，`read(fd)` 的 fd 是这个表的下标 |
| **I/O 等待队列** | 进程等磁盘时进入 BLOCKED，I/O 完成后被中断唤醒回 READY |
| **管道 / IPC** | 进程间通信（管道、消息队列、共享内存、socket） |

### 【次要缺失，偏实现细节】

| 缺失内容 | 说明 |
|---------|------|
| **多级页表** | 真实 OS（64位）是四级页表，MiniOS 用一维数组模拟 |
| **TLB** | 页表缓存，加速地址翻译，缺失会让虚拟内存慢几百倍 |
| **文件权限检查** | `rwxr-xr-x` 字段存了但从未实际校验 |
| **日志（Journal）** | ext4 等文件系统崩溃后能恢复，靠写日志实现 |
| **网络栈** | TCP/IP、socket，MiniOS 完全没有 |
| **设备驱动** | 键盘、屏幕、网卡等驱动层 |
| **Bootloader** | OS 如何从 BIOS 启动加载到内存 |

## 后续可扩展的方向

完成六个阶段后，可以按以下顺序扩展：

### 第七阶段：用户态/内核态模拟
```
新增：kernel/syscall.h/cpp
  sys_read(fd, buf, size)   # 唯一合法的读文件方式
  sys_write(fd, buf, size)  # 唯一合法的写文件方式
  sys_fork()                # 创建进程
  sys_exit()                # 进程退出

新增 PCB 字段：
  ring (0=内核态, 3=用户态)
  file_descriptor_table[]   # 每个进程的打开文件表
```

### 第八阶段：Page Cache + 块设备桥接
```
新增：memory/page_cache.h/cpp
  struct Page { int block_num; string data; bool dirty; }
  get(block_num)   # 命中返回，未命中走磁盘
  evict()          # 缓存淘汰（复用第六阶段 LRU）

新增：io/block_device.h/cpp
  logical_to_physical(block_num) → sector_num
  submit_io_request(sector_num)  → 发给 DiskScheduler

改造：FileSystem::read()
  不再直接返回 content，先查 Page Cache
```

**效果：** 文件系统和磁盘调度真正连通：
```
进程: sys_read("log.txt")
  → FileSystem: 找 inode，得到块号 [42, 43]
  → PageCache: 块 42 未命中
  → BlockDevice: 块42 → 物理扇区 84000
  → DiskScheduler: SCAN 调度，模拟寻道
  → 数据回到 PageCache，返回给进程
```

### 第九阶段（进阶）：真实多进程并发
```
每个"用户进程"用真实的 std::thread 运行
内核调度器在线程之间做真正的上下文切换
IPC：管道、共享内存
```

## 一句话总结

```
MiniOS 实现的是：OS 的"是什么"和"为什么"（算法 + 数据结构）
真实 OS 还需要：OS 的"怎么运行起来"（中断驱动 + 特权模式 + 真实硬件）
```

MiniOS 覆盖了《操作系统（第六版）》绝大部分算法和数据结构，这正是教材的考察重点。  
中断机制和用户态/内核态属于计算机体系结构的范畴，是计组/OS底层的内容。
